# Feng 语言变量绑定与作用域规范

本文档补充 [feng-language.md](./feng-language.md) 中关于 `let` / `var` 的概要说明,聚焦 Feng 语言中的变量绑定、参数可变性和块作用域规则。

## 1 职责

`let` / `var` 是 Feng 语言中声明绑定方式的机制,承担以下职责:

- 通过 `let` 声明不可变绑定,通过 `var` 声明可变绑定,在语法层面直接区分只读与可写语义。
- 覆盖所有声明位置: 模块内变量、函数内局部变量、函数参数和 `type` / `@fixed type` 成员变量。
- 通过块作用域规则约束绑定的可见范围与名称屏蔽行为。

## 2 术语

- 不可变绑定: 通过 `let` 声明的绑定; 初始化后不可再次赋值。
- 可变绑定: 通过 `var` 声明的绑定; 可在当前作用域内重新赋值。
- 默认零值: 无初始值时,编译器按对应类型规则自动赋予的初始值。
- 块作用域: 绑定的可见范围由其声明所在的块及其子块确定。
- 名称屏蔽: 子级块中声明的同名变量优先于外层同名绑定; 子级块结束后外层绑定重新可见。

## 3 语法

正确语法一,基本 `let` / `var` 声明:

```feng
// 不可变绑定,有初始值,省略类型
let name = "feng";

// 可变绑定,有初始值,省略类型
var count = 0;
count = count + 1;

// 无初始值,必须显式标注类型
let age: int;
var total: int;
```

正确语法二,模块级与函数内局部变量:

```feng
pu mod demo.binding;

let app_name = "feng";
var total: int;

fn main(args: string[]) {
    let user = "guest";
    var retry = 0;
    retry = retry + 1;
}
```

正确语法三,函数参数声明:

```feng
// 省略 let / var,默认等价于 let 参数
fn sum(a: int, b: int): int {
    return a + b;
}

// 显式 var 参数,函数体内可修改
fn increase(step: int, var total: int) {
    total = total + step;
}

// 显式 let 参数
fn visit(let name: string) {
    print(name);
}
```

正确语法四,`type` 与 `@fixed type` 成员声明:

```feng
type User {
    var name: string;
    pr let id: int;
}

@fixed
type Point {
    var x: int;
    var y: int;
}
```

正确语法五,块作用域与名称屏蔽:

```feng
fn show_scope(name: string) {
    let label = "outer";

    if name != "" {
        var label = "inner";
        label = label + "-block";
        print(label);
    }

    print(label);
}
```

错语法一,模块内变量省略 `let` / `var`:

```feng
// 错误: 模块内变量必须显式使用 let 或 var
app_name = "feng";
total: int;
```

错语法二,函数参数省略类型:

```feng
// 错误: 参数即使省略 let / var 也不能省略类型
fn sum(a, b): int {
    return a + b;
}
```

## 4 语义

- `let` 声明只读绑定,适合不会在后续流程中被重新赋值的值; `var` 声明可写绑定,适合需要在作用域内更新状态的值。
- 两者都只是绑定规则的声明方式,不改变变量的类型检查、类型推导或默认零值规则。
- 有初始值时,`let` / `var` 均可省略类型标注,由编译器自动推导; 无初始值时均需显式标注类型,再按对应类型执行默认零值初始化。
- 函数参数省略 `let` / `var` 时,默认等价于 `let` 参数,函数体内不可直接修改其值; 如需修改,必须显式写为 `var` 参数。
- `Lambda` 参数沿用相同规则: 可省略 `let` / `var`,省略时默认按 `let` 处理。
- `let` 与 `var` 都遵循块作用域规则; 子级块中声明同名变量时,编译器优先解析到最近作用域中的绑定,从而屏蔽外层同名变量; 子级块结束后外层被屏蔽的同名变量重新可见。
- 是否允许重新赋值只由当前实际引用到的绑定是 `let` 还是 `var` 决定。
- 模块级 `let` / `var` 可与 `pu` / `pr` 组合使用; 公开模块中的公开模块级绑定可被其他模块通过 `use` 引入; 被引入后按其原始 `let` / `var` 语义进入使用方可见作用域,不参与函数重载。
- `type` / `@fixed type` 成员的 `let` / `var` 同样可与 `pu` / `pr` 访问控制组合使用; 成员的可变性由成员自身的 `let` / `var` 决定,与可见性控制相互独立。

## 5 规则

分为「必须、禁止、建议」。

- [必须] 模块内变量、函数内局部变量和 `type` / `@fixed type` 成员变量必须显式书写 `let` 或 `var`,不可省略; 是否可省略类型取决于是否提供初始值,与 `let` / `var` 的必写要求无关。
- [必须] 函数参数必须显式标注类型; 绑定方式可省略,省略时默认等价于 `let`。
- [必须] 有初始值时可省略类型标注,由编译器推导; 无初始值时必须显式标注类型,编译器自动赋予默认零值。
- [必须] `let` 绑定初始化之后不可再次赋值。
- [必须] `var` 绑定允许在当前作用域内重新赋值。
- [必须] 子级块可以重新声明与外层同名的 `let` 或 `var` 绑定; 名称冲突时最近作用域中的绑定优先。
- [禁止] 模块内变量或函数内局部变量省略 `let` / `var`。
- [禁止] 函数参数省略类型标注。
- [建议] 优先使用 `let`,仅在确实需要更新状态时才使用 `var`。

## 6 编译期

- 编译器必须检查所有声明位置(函数参数除外)是否显式使用了 `let` 或 `var`。
- 编译器必须检查 `let` 绑定在初始化后是否发生重新赋值,发现则报错。
- 编译器必须检查无初始值的绑定是否显式标注了类型。
- 编译器必须检查函数参数是否标注了类型。
- 编译器必须按块作用域规则解析名称,确保同名绑定的屏蔽关系正确。
- 编译器必须检查 `type` / `@fixed type` 成员中 `let` / `var` 的绑定合法性,并遵循 `feng-type.md` 中的成员绑定规则。

## 7 运行时

- 无初始值的绑定在运行时按类型规则得到默认零值; 不存在未初始化的野值。
- `let` 绑定在整个生命周期内保持第一次绑定的值不变。
- `var` 绑定的值可在作用域内被更新。
- 块作用域结束时,该块内声明的所有绑定生命周期结束; 被屏蔽的外层绑定重新可见。
- 模块级绑定的生命周期与模块的加载周期一致。
