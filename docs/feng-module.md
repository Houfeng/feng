# Feng 语言模块规范

本文档用于补充 [feng-language.md](./feng-language.md) 中的模块系统概要说明,聚焦 feng 语言的模块声明、模块导入与模块使用规则。

## 1 模块系统概览

- Feng 使用 `mod` 进行文件级模块声明,使用 `use` 导入外部模块或二进制包。
- 模块系统支持多级命名空间,通过 `.` 进行层级分隔。
- 模块声明与模块导入在文件内有固定位置约束。
- 模块具备可见性控制,默认包外不可见,需显式声明公开访问权限后才可被外部导入。
- 模块既可用于源码组织,也可用于 feng 自有二进制包和 C ABI 兼容包的导入。

## 2 模块声明规则

- `mod` 必须是文件中的第一个非空且非注释行,无花括号、无代码块。
- 一个文件只能属于一个 `mod`; 同一个 `mod` 可以分布在多个文件中,编译后自动合并。
- 模块名支持 `.` 分隔多级命名空间。
- 同一个 `mod` 在单文件或多文件合并后,模块级 `type` 名称必须唯一,模块级 `fn` 名称也必须唯一; 若出现重名,编译期报错。

重名约束说明:

- 若同一模块在多个文件中分别声明同名 `type`,编译期报错。
- 若同一模块在多个文件中分别声明同名顶层 `fn`,编译期报错。
- 上述约束在单文件内同样成立,不能通过拆分文件规避。

## 3 模块可见性

- 模块可见性遵循 `pr` / `pu` 规则。
- `mod` 默认等价于 `pr mod`,包外不可见。
- 需要对外暴露时,必须显式使用 `pu mod` 声明公开模块,外部代码才可导入。

可见性示例:

默认私有模块:

```feng
mod app.internal.cache;
```

显式私有模块:

```feng
pr mod app.internal.cache;
```

公开模块:

```feng
pu mod app.api.user;
```

导入公开模块:

```feng
use app.api.user;
```

## 4 模块导入规则

- `use` 必须位于模块声明之后、类型与函数定义之前。
- 调用外部成员格式: `模块名.成员名`。
- 支持导入源码模块、feng 自有二进制包、C ABI 兼容包。
- 支持使用 `use 模块名 as 别名` 为导入目标声明文件内别名。

别名导入规则:

- `as` 别名仅在当前文件内生效,不改变被导入模块的真实模块名。
- 使用别名后,调用外部成员格式为 `别名.成员名`。
- 同一文件内的 `use` 别名必须唯一,不能与其他 `use` 别名重名。
- 若未使用 `as`,则默认仍以完整模块名访问成员。

```feng
use my.app.user as user;
use my.utils.math as math;

let item = user.get_current();
let total = math.add(1, 2);
```

## 5 模块级初始化顺序

模块级 `let` / `var` 变量若带有初始化表达式,则属于模块级初始化逻辑的一部分。模块级初始化在程序启动时自动执行,每个模块只初始化一次。

规则说明:

- 单个文件内的模块级变量按自上而下顺序初始化。
- 同一 `mod` 分布在多个文件中时,先按文件的标准化相对路径字典序排序,再分别按各文件内的自上而下顺序执行初始化。
- 若模块 `A` 通过 `use` 依赖模块 `B`,则 `B` 的模块级初始化先于 `A` 执行。
- 同一文件内多个 `use` 声明按书写顺序建立依赖初始化顺序。
- 模块初始化完成后,后续对该模块成员的访问不会再次触发初始化。
- 若模块级初始化之间形成循环依赖,编译期报错。

```feng
pu mod app.main;

use app.config;
use app.runtime;

let name = app.config.load_name();
let boot = app.runtime.start(name);
```

在上例中,初始化顺序为:

1. 先初始化 `app.config`
2. 再初始化 `app.runtime`
3. 最后按当前文件自上而下初始化 `name` 与 `boot`

## 6 书写顺序与格式建议

- `mod` 声明后进入 `use` 导入段时,建议在两者之间保留一个空行。
- 多个 `use` 声明之间不空行,保持连续排列。
- `use` 导入段与下方类型、函数或其他实现代码之间建议保留一个空行。

书写示例:

```feng
pu mod my.app.user;

use my.utils;
use my.core;
use my.app.service as service;

type User {
    pu var name: string;
}

fn load() {
    service.run();
}
```
