# Feng 语言模块系统规范

本文档用于补充 [feng-language.md](./feng-language.md) 中的模块系统概要说明,聚焦 feng 语言的模块声明、模块导入与模块使用规则。

## 1 模块系统概览

- Feng 使用 `mod` 进行文件级模块声明,使用 `use` 导入外部模块或二进制包。
- 模块系统支持多级命名空间,通过 `.` 进行层级分隔。
- 模块声明与模块导入在文件内有固定位置约束。
- 模块具备可见性控制,默认包外不可见,需显式声明公开访问权限后才可被外部导入。
- 模块既可用于源码组织,也可用于 feng 自有二进制包和 C ABI 兼容包的导入。

## 2 模块声明规则

- `mod` 必须是文件中的第一个非空且非注释行,无花括号、无代码块。
- 一个文件只能属于一个 `mod`; 同一个 `mod` 可以分布在多个文件中,编译后自动合并。
- 模块名支持 `.` 分隔多级命名空间。

## 3 模块可见性

- 模块可见性遵循 `pr` / `pu` 规则。
- `mod` 默认等价于 `pr mod`,包外不可见。
- 需要对外暴露时,必须显式使用 `pu mod` 声明公开模块,外部代码才可导入。

可见性示例:

默认私有模块:

```feng
mod app.internal.cache;
```

显式私有模块:

```feng
pr mod app.internal.cache;
```

公开模块:

```feng
pu mod app.api.user;
```

导入公开模块:

```feng
use app.api.user;
```

## 4 模块导入规则

- `use` 必须位于模块声明之后、类型与函数定义之前。
- 调用外部成员格式: `模块名.成员名`。
- 支持导入源码模块、feng 自有二进制包、C ABI 兼容包。

## 5 书写顺序与格式建议

- `mod` 声明后进入 `use` 导入段时,建议在两者之间保留一个空行。
- 多个 `use` 声明之间不空行,保持连续排列。
- `use` 导入段与下方类型、函数或其他实现代码之间建议保留一个空行。

书写示例:

```feng
pu mod my.app.user;

use my.utils;
use my.core;

type User {
    pu var name: string;
}
```
