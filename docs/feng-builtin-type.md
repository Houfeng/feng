# Feng 内建类型规范

本文档补充 [feng-language.md](./feng-language.md) 中的类型系统概要，描述 Feng 语言的内建类型：有符号整数、无符号整数、浮点数、布尔、字符串与基础数组。这些类型由语言直接提供，无需 import，其默认零值、类型别名与核心语义均在本文档中规定。

## 1 职责

- 提供最基本的数值、布尔、字符串与数组能力，作为所有用户自定义类型的基础。
- 通过类型别名（如 `int`、`long`、`byte`、`float`、`double`）为常用整数与浮点类型提供可读性更强的书写形式。
- 规定各类型的默认零值、C 等价类型与整数溢出策略，并定义字符串与基础数组的核心语义。

## 2 语义

- 基础值类型包含有符号整数、无符号整数、浮点与布尔; 部分类型提供别名以方便书写: `i32` → `int`、`i64` → `long`、`u8` → `byte`、`f32` → `float`、`f64` → `double`; 别名与原类型完全等价,不引入新类型; 各类型的 C 等价类型与默认零值见 §3。
- `string` 的默认零值是 `""`; 数组使用 `T[]` 语法,默认零值是 `[]`,并由自动内存管理机制托管; `string` 与数组在 C ABI 边界上的兼容资格、桥接与限制统一见 [Feng 语言 C 互操作规范](./feng-interop.md)。
- C 指针类型使用 `*T` 语法,默认零值为空指针,仅用于 C 互操作。
- Feng 的类型转换一律显式书写; 语言不提供隐式类型转换。
- 数值字面量的默认类型: 未显式声明类型时，整数字面量默认推导为 `int`（即 `i32`），浮点字面量默认推导为 `double`（即 `f64`）；如需其他精度或宽度，必须显式标注类型（如 `let x: i64 = 100;` 或 `let y: f32 = 1.5;`）。
- 整数类型的溢出策略: 编译期可确定的溢出（如字面量赋值超出目标类型范围）视为编译错误; 运行时整数运算溢出按对应位宽静默回绕，有符号与无符号均适用，语言不提供运行时溢出检测。

## 3 内置基础类型

| Feng 类型 | 别名 | 对应 C 类型 | 说明 | 默认零值 |
| --- | --- | --- | --- | --- |
| `i8` | - | `int8_t` | 8 位有符号整数 | `0` |
| `i16` | - | `int16_t` | 16 位有符号整数 | `0` |
| `i32` | `int` | `int32_t` | 32 位有符号整数 | `0` |
| `i64` | `long` | `int64_t` | 64 位有符号整数 | `0` |
| `u8` | `byte` | `uint8_t` | 8 位无符号整数 | `0` |
| `u16` | - | `uint16_t` | 16 位无符号整数 | `0` |
| `u32` | - | `uint32_t` | 32 位无符号整数 | `0` |
| `u64` | - | `uint64_t` | 64 位无符号整数 | `0` |
| `f32` | `float` | `float` | 单精度浮点数 | `0.0` |
| `f64` | `double` | `double` | 双精度浮点数 | `0.0` |
| `bool` | - | `bool` | 布尔类型,值为 `true` / `false` | `false` |
| `string` | - | - | UTF-8 字符串 | `""` |

## 4 字符串类型

`string` 是 Feng 的内置字符串类型，采用 UTF-8 编码，由自动内存管理机制托管，内容不可变。

补充定义:

- `string` 在语言实现层是受托管的引用类型；语言层 `string` 值表示为指向运行时字符串对象的引用，而不是内联字节序列。
- `string` 对用户表现为值语义：赋值、传参与返回复制的是引用，不复制字符串字节内容；由于字符串内容不可变且 `==` / `!=` 按内容比较，用户不可观察到共享可变状态。
- `string` 的逻辑长度按 UTF-8 字节数定义，而不是 Unicode 字符数或字素簇数。
- 当前运行时会为字符串对象额外保留一个结尾 `0` 字节，便于运行时内部与 C ABI 字符串桥接；该字节不属于字符串内容，不计入逻辑长度。跨越 C ABI 边界时的具体处理由编译器和运行时自动完成。当前运行时对象布局与释放规则见 [Feng 语言对象生命周期规范](./feng-lifetime.md)。

语法规则:

- 字符串字面量必须使用双引号 `"` 包裹，不支持单引号。
- 不支持多行字符串字面量；换行需使用转义序列 `\n`。
- 支持以下转义序列:

| 转义序列 | 含义 |
| --- | --- |
| `\\` | 反斜杠 `\` |
| `\"` | 双引号 `"` |
| `\n` | 换行符 |
| `\r` | 回车符 |
| `\t` | 水平制表符 |
| `\0` | 空字符 |

语义规则:

- `string` 是不可变类型；变量绑定的可变性（`let`/`var`）控制的是变量能否重新绑定到新字符串，而非修改字符串内容本身。
- `string` 的 `==` / `!=` 按内容（值语义）比较，不依赖底层引用身份。
- `string` 支持 `+` 运算符进行拼接，结果为新的 `string` 值。
- `string` 在 C ABI 边界上的兼容资格、桥接与限制统一见 [Feng 语言 C 互操作规范](./feng-interop.md)。`string` 本身没有 `char*` 之类的类型别名。

```feng
let greeting = "Hello, World!";
let line = "第一行\n第二行";
let path = "C:\\Users\\feng";
let msg = "say: \"hi\"";
```

## 5 数组

`T[]` 与 `T[]!` 是 Feng 的内置数组类型。Feng 不额外提供独立的“动态数组”类型；数组实例在初始化完成后长度固定不变。二者都采用引用语义：赋值、传参与返回复制的是数组引用，而不是数组元素本体。

数组的细节定义、可写性规则、类型兼容与显式转换均以本节为准；数组在 C ABI 边界上的兼容资格与桥接规则统一见 [Feng 语言 C 互操作规范](./feng-interop.md)。其他文档仅在各自语境中引用本节，不重复展开数组细则。

补充定义:

- `T[]` 表示元素类型为 `T` 的只读数组类型；`T[]!` 表示该数组层的元素可写。多维数组沿 `[]` 递进书写，`!` 只约束其紧邻的 `[]` 这一层，因此可逐层组合写成 `T[][]`、`T[][]!`、`T[]![]`、`T[]![]!` 等形式。
- 对多维数组而言，每一层 `[]` 的可写性独立判断。`T[][]!` 表示最外层数组元素可写、其元素类型仍是只读 `T[]`；`T[]![]` 表示最外层数组元素不可写，但其元素类型是可写数组 `T[]!`；`T[]![]!` 表示两层都可写。
- 数组类型表达式中不包含长度；数组长度是运行时属性，而不是静态类型的一部分。元素类型、数组层数以及每一层 `[]` 对应的可写标记一致的数组才视为同一数组类型，不因实例长度不同而区分为不同类型。
- 数组实例在创建完成后长度固定；当前版本不提供修改数组长度的语言机制。`[]` 表示长度为 `0` 的只读空数组，`[]!` 表示长度为 `0` 的元素可写空数组。
- `T[]` 的当前层元素默认不可变；当前版本不提供通过该层下标位置原地改写其元素的语言机制。对多维数组同理，是否可在某一层通过下标写入，取决于该层 `[]` 是否标记 `!`。
- `T[]!` 的当前层元素可通过该层下标位置原地改写。对多维数组同理，`!` 只赋予其紧邻 `[]` 这一层的写入能力，不会自动传播到其他层级。数组的可写性只影响对应层的元素写入能力，不改变数组长度固定与引用语义。
- 数组类型之间默认不隐式兼容。显式转换只允许在保持元素类型与数组层数不变的前提下移除某一层或多层已有的 `!` 标记，不允许增加 `!` 标记。例如允许从 `T[]!` 显式转换到 `T[]`、从 `T[][]!` 显式转换到 `T[][]`、从 `T[]![]` 显式转换到 `T[][]`，不允许反向转换。
- 数组在语言实现层是受托管的引用类型；多个变量可引用同一数组实例。
- 数组变量绑定的可变性（`let`/`var`）控制的是绑定能否重新指向另一个数组，而不是数组长度或元素可写性。
- 数组在 C ABI 边界上的兼容资格、桥接与限制统一见 [Feng 语言 C 互操作规范](./feng-interop.md)。

```feng
let empty: int[] = [];
let nums = [1, 2, 3];
let writable_empty: int[]! = []!;
let writable_nums = [1, 2, 3]!;
let outer_writable_matrix: int[][]! = [[1, 2], [3, 4]]!;
let inner_writable_matrix: int[]![] = [[1, 2]!, [3, 4]!];
let fully_writable_matrix: int[]![]! = [[1, 2]!, [3, 4]!]!;
let alias = nums;
var current = nums;
current = [4, 5, 6];
writable_nums[0] = 10;
outer_writable_matrix[0] = [9, 10];
inner_writable_matrix[0][0] = 9;
fully_writable_matrix[0] = [5, 6]!;
fully_writable_matrix[0][0] = 7;

let readonly_nums = (int[])writable_nums;
let readonly_outer_matrix = (int[][])outer_writable_matrix;
let readonly_inner_matrix = (int[][])inner_writable_matrix;

// nums[0] = 10; // 非法: 基础数组元素默认不可变
// outer_writable_matrix[0][0] = 9; // 非法: 内层 `T[]` 未标记 `!`
// inner_writable_matrix[0] = [9, 10]!; // 非法: 最外层 `[]` 未标记 `!`
// let writable_again = (int[]!)nums; // 非法: 不允许从只读数组显式转换到可写数组
```

## 6 规则

分为「必须」。

- [必须] 未显式声明类型时，整数字面量默认推导为 `int`（`i32`），浮点字面量默认推导为 `double`（`f64`）；如需其他类型，必须显式标注。
- [必须] Feng 的类型转换一律显式书写；语言不提供隐式类型转换。
- [必须] 整数字面量赋值到具体整数类型时，若字面量值超出该类型的有效范围，编译期报错。
- [必须] `string` 的逻辑长度按 UTF-8 字节数定义；运行时为桥接 C 额外保留的结尾 `0` 字节不属于字符串内容，也不计入逻辑长度。
- [必须] 数组类型表达式中不包含长度；判断数组类型兼容性时，长度不是静态类型的一部分。
- [必须] 可写数组层通过在对应的 `[]` 后追加 `!` 表示，例如 `T[][]!`、`T[]![]`、`T[]![]!`。
- [必须] `!` 只约束其紧邻的 `[]` 这一层；未标记 `!` 的数组层，其元素不可通过该层下标位置直接赋值修改。
- [必须] 元素类型、数组层数和各对应数组层的可写标记一致的数组才视为同一数组类型，不因实例长度不同而区分为不同类型。
- [必须] 基础数组实例一旦创建完成，其长度保持不变。
- [必须] `T[]` 的当前层元素默认不可变；当前版本不得通过该层下标位置直接赋值修改元素。
- [必须] `T[]!` 的当前层元素可通过该层下标位置直接赋值修改。
- [必须] 数组类型之间默认不隐式兼容；显式转换只允许移除某一层或多层已有的 `!` 标记，不允许增加 `!` 标记。
- [必须] 数组绑定的 `let` / `var` 仅控制绑定能否重新指向另一数组，不改变数组长度固定以及由类型决定的元素可写性语义。
- [必须] `string` 与数组在 C ABI 边界上的兼容资格、桥接与限制统一遵循 [Feng 语言 C 互操作规范](./feng-interop.md)。

## 7 编译期

- 编译器必须在类型推导阶段将无显式类型标注的整数字面量推导为 `int`（`i32`），将无显式类型标注的浮点字面量推导为 `double`（`f64`）。
- 编译器必须检查整数字面量（含十六进制 `0x`、二进制 `0b`、八进制 `0o` 等各进制形式）是否超出目标整数类型的有效范围，超出时编译期报错。
- 编译器必须将 `string` 视为内建托管引用类型；语言层 `string` 值的实现表示是对运行时字符串对象的引用，而不是内联值布局。
- 编译器必须保持 `string` 的值语义：`==` / `!=` 的语义为按内容比较；实现可将“同引用”与“长度不等”作为快路径，但不得把引用身份当作相等语义本体。
- 编译器必须将数组视为内建托管引用类型；语言层数组值的实现表示是对运行时数组对象的引用，而不是内联元素序列。
- 编译器必须将数组长度视为运行时属性而非静态类型的一部分；判断数组类型兼容性时，必须比较元素类型、数组层数以及各对应数组层的可写标记。
- 编译器必须把每个 `[]!` 解释为其紧邻数组层的可写标记；未标记 `!` 的数组层默认不可写。
- 编译器必须允许在多维数组类型的任意数组层使用 `[]!` 标记，并拒绝任何未紧跟在 `[]` 之后的孤立 `!` 用法。
- 编译器必须拒绝任何对 `T[]` 下标位置的直接赋值。
- 编译器必须允许对 `T[]!` 下标位置的直接赋值,并检查被写入值的元素类型兼容性。
- 编译器必须拒绝任何会增加数组层 `!` 标记的显式转换；必须允许在保持元素类型与数组层数不变的前提下移除一个或多个已有 `!` 标记的显式转换。
- 编译器必须按 [Feng 语言 C 互操作规范](./feng-interop.md) 校验 `string` 与数组在 C ABI 边界上的兼容资格与桥接限制。

## 8 运行时

- 整数运算在运行时发生溢出时，按对应位宽静默回绕，有符号与无符号类型均适用; 语言不提供运行时溢出检测或异常抛出。
- `string` 在运行时以受托管堆对象表示；对象内容创建完成后不可原地修改，任何结果不同的字符串操作都产生新的 `string` 值。当前对象布局与释放规则见 [Feng 语言对象生命周期规范](./feng-lifetime.md)。
- 数组在运行时以受托管堆对象表示；赋值、传参与返回默认复制数组引用而非元素内容。数组对象一旦创建完成，其长度保持不变；各数组层的元素是否可原地改写，按该层 `[]` 是否标记 `!` 独立判断。当前对象布局与释放规则见 [Feng 语言对象生命周期规范](./feng-lifetime.md)。
- `string` 与数组传递到 C ABI 边界时,其桥接与必要的生命周期协调统一遵循 [Feng 语言 C 互操作规范](./feng-interop.md)。
