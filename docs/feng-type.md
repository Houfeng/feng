# Feng 语言类型规范

本文档补充 [feng-language.md](./feng-language.md) 中的类型系统概要,聚焦具体类型定义、对象行为模型、初始化与契约协作约束。

## 1 职责

`type` 是 Feng 语言中用于声明具体类型实现的机制,承担以下职责:

- 定义具体数据结构与行为实现,声明成员字段与构造函数,确立实例化与初始化语义。
- 定义类型成员绑定点,通过成员上的 `let`/`var` 决定成员可变性与初始化约束。
- 通过 `@fixed` 标注,声明适合进入 C ABI 边界的固定布局类型。
- 作为契约实现者,与 `spec`/`fit` 协作建立“具体类型满足契约”的关系。

## 2 术语

- 具名类型: 通过 `type` 声明的类型。
- 对象类型: 以对象语法声明的具名类型(如 `type User { ... }`); 是否标注 `@fixed` 不改变其类型命名,只影响 ABI 边界约束。
- 对象字面量: 以 `Type() { ... }`、`Type() {}`、`Type(args) { ... }`、`Type(args) {}`、`Type { ... }` 或 `Type {}` 形式进行的对象初始化表达式; 其中 `Type { ... }` 是 `Type() { ... }` 的简写,`Type {}` 是 `Type() {}` 的简写。
- 数组类型: 使用 `T[]` 语法声明的动态数组类型,可递进写成 `T[][]`、`T[][][]` 等多维形式。
- C 指针类型: 使用 `*T` 语法声明、仅用于 C 互操作的指针类型。
- ABI 约束: 类型标注 `@fixed` 后,在 C ABI 边界需要满足的固定布局与使用约束。
- 成员绑定: `type` 成员上的 `let`/`var` 绑定声明,用于定义成员可变性与赋值约束。
- 默认零值: 无初始值绑定在编译期按类型规则得到的默认值。
- 方法值: 以 `obj.method` 形式获取的、已绑定到特定对象实例的可调用值; 调用时 `self` 始终指向该实例,不会因传递或别名而改变。

## 3 语法

正确语法一,基础与对象类型:

```feng
type User {
  let id: int;
  var name: string;

  fn User(id: int, name: string) {
    self.id = id;
    self.name = name;
  }
}
```

正确语法二,带 `@fixed` 的对象类型:

```feng
@fixed
type Point {
  var x: int;
  var y: int;
}
```

正确语法三,显式无参构造与对象字面量简写:

```feng
type Counter {
  let value: int;

  fn Counter() {
    self.value = 1;
  }
}

let a = Counter();
let b = Counter() {};
let c = Counter {};
let d = User(1, "feng") {};
let e = User(2, "feng") { name: "Feng" };
```

错语法一,在类型内嵌套声明类型:

```feng
type A {
  type B {
    var x: int;
  }
}
```

错语法二,成员类型位置内联匿名类型:

```feng
type A {
  var c: {
    var c1: string;
    var c2: string;
  };
}
```

错语法三,无参构造不可见时使用对象字面量简写:

```feng
type Hidden {
  pr fn Hidden() {}
}

let x = Hidden {};
```

正确语法四,成员方法与 `self`:

```feng
type Counter {
  var value: int;

  fn inc() {
    self.value = self.value + 1;
  }
}
```

正确语法五,方法值与重载决议:

```feng
type User {
  fn say() {}
  fn say(msg: string) {}
}

spec M0(): void;

fn test(m: M0) {
  m();
}

let u = User {};
test(u.say);
let s_ok: M0 = u.say;
let s = u.say; // 错误: say 存在多个重载,当前上下文无法推导 s 的函数类型
```

`u.say` 在作为参数传给 `test` 时,传递的是一个已绑定到 `u` 所引用实例的方法值; 由于目标类型 `M0` 能唯一确定无参重载,`test` 内部执行 `m()` 时,`say` 中的 `self` 仍然指向该 `User` 实例。`let s_ok: M0 = u.say;` 同样合法,因为显式声明的目标函数类型足以唯一选定无参重载。`let s = u.say;` 则因缺少足以唯一选定重载的目标函数类型,编译期报错。

正确语法六,引用别名:

```feng
type User {
  var name: string;
}

let a = User { name: "guest" };
let b = a;
b.name = "admin";  // a.name 同时变为 "admin"
```

正确语法七,`let`/`var` 两层语义:

```feng
type User {
  var name: string;
  let created_at: int;
}

let user = User { name: "guest", created_at: 1 };
user.name = "admin";    // 合法: name 是 var 成员
// user.created_at = 2; // 非法: let 成员初始化后不可再赋值
// user = User {};      // 非法: user 绑定为 let,不可重新指向另一个对象
```

## 4 语义

- Feng 采用强类型、静态类型系统,无 `any` 类型,变量、参数与返回值在编译期完成类型检查。
- 基础值类型包含有符号整数、无符号整数、浮点与布尔; `i64` 的别名是 `int`,`f64` 的别名是 `float`; 各类型的 C 等价类型与默认零值见 §4.1。
- `string` 的默认零值是 `""`; 动态数组使用 `T[]` 语法,默认零值是 `[]`,并由自动内存管理机制托管; 二者都不可直接用于 C 互操作。
- C 指针类型使用 `*T` 语法,默认零值为空指针,仅用于 C 互操作。
- 统一使用 `type` 定义具名具体类型; `spec` 负责定义契约形状与可调用形状。
- `@fixed` 只引入 ABI 边界约束,不改变类型本身的命名、语义分类与 `type` 语法形式; 合法性由语义分析按 ABI 规则检查。
- 具名类型声明必须位于模块级顶层,不支持嵌套声明、匿名类型定义或在类型位置直接内联声明类型。
- 同一模块内的具名具体类型允许前向引用与互引用。
- 对象类型支持 `Type()`、`Type(args)`、`Type() { ... }`、`Type() {}`、`Type(args) { ... }` 与 `Type(args) {}` 等对象构造写法; `Type { ... }` 与 `Type {}` 分别是 `Type() { ... }` 与 `Type() {}` 的简写。
- 构造对象实例的所有写法都遵循"成员声明及绑定初值 -> 构造函数 -> 对象字面量初始化"的统一流程; 某些写法只是省略了空的对象字面量阶段或把无参对象字面量写成简写形式。
- 在成员声明及绑定初值阶段,有初始值绑定的成员完成显式绑定; 无初始值的成员底层存储先取类型默认零值,但这不视为该成员已完成显式绑定,后续仍可在构造函数或对象字面量阶段完成首次显式绑定。
- `var` 成员在三个构造阶段内及创建完成后均可多次赋值,不受单次最终绑定限制。
- 构造函数中的 `self` 指向当前正在初始化的对象实例; 成员方法中的 `self` 指向当前调用该方法的对象实例; 两种情况下 `self` 的绑定规则相同,均由编译器隐式提供。
- `type` 及 `fit` 块的成员 `fn` 与构造函数中可使用 `self` 引用当前实例; `self` 由编译器隐式提供,无需在参数列表中显式声明; 该绑定始终稳定,不会因别名赋值、参数传递或作为方法值使用而改变; 普通 `fn` 或 `Lambda` 中直接使用 `self` 编译期报错。
- `let`/`var` 在对象场景中分为两层语义: 绑定层——变量名是否允许重新绑定到另一个对象; 成员层——对象的具体成员是否允许被改写; `let user = ...;` 表示 `user` 本身不可重新绑定,但不自动冻结对象内部的 `var` 成员。
- 成员方法通过 `obj.method(args)` 调用; 以 `obj.method` 形式获取方法值时,该值已绑定到 `obj` 引用的实例,`self` 不会因传递或别名改变; 同名方法有多个重载时,赋值目标须提供足以唯一选定重载的函数类型,否则编译期报错。
- 普通 `type` 不可直接出现在 `@fixed fn` 的参数或返回值位置,也不应成为 `@fixed type` 的直接依赖成员; `@fixed type` 的方法不改变实例布局,其 ABI 资格由方法自身是否标注 `@fixed` 决定。
- 整数类型的溢出策略: 编译期可确定的溢出（如字面量赋值超出目标类型范围）视为编译错误; 运行时整数运算溢出按对应位宽静默回绕，有符号与无符号均适用，语言不提供运行时溢出检测。

### 4.1 内置基础类型

| Feng 类型 | 别名 | 对应 C 类型 | 说明 | 默认零值 |
| --- | --- | --- | --- | --- |
| `i8` | - | `int8_t` | 8 位有符号整数 | `0` |
| `i16` | - | `int16_t` | 16 位有符号整数 | `0` |
| `i32` | - | `int32_t` | 32 位有符号整数 | `0` |
| `i64` | `int` | `int64_t` | 64 位有符号整数 | `0` |
| `u8` | - | `uint8_t` | 8 位无符号整数 | `0` |
| `u16` | - | `uint16_t` | 16 位无符号整数 | `0` |
| `u32` | - | `uint32_t` | 32 位无符号整数 | `0` |
| `u64` | - | `uint64_t` | 64 位无符号整数 | `0` |
| `f32` | - | `float` | 单精度浮点数 | `0.0` |
| `f64` | `float` | `double` | 双精度浮点数 | `0.0` |
| `bool` | - | `bool` | 布尔类型,值为 `true` / `false` | `false` |

## 5 规则

分为「必须、禁止、建议」。

- [必须] 有初始值绑定可省略类型并由编译器推导; 无初始值绑定必须显式标注类型并使用默认零值初始化。
- [必须] 对象类型成员必须使用 `let` 或 `var` 修饰。
- [必须] 类型成员默认公开; 需要收窄访问范围时,必须显式使用 `pr`。
- [必须] 成员 `let`/`var` 绑定语义与赋值规则遵循 `feng-binding.md` 的统一绑定规范。
- [必须] 对象类型成员类型必须引用已声明具名类型。
- [必须] 构造函数与类型同名,按“名称 + 参数个数 + 参数类型”参与重载决议。
- [必须] 对象类型支持 `Type()`、`Type(args)`、`Type() { ... }`、`Type() {}`、`Type(args) { ... }` 与 `Type(args) {}`; 其中 `Type { ... }` 与 `Type {}` 分别是 `Type() { ... }` 与 `Type() {}` 的简写。
- [必须] 构造对象实例的所有写法都必须遵循“成员声明及绑定初值 -> 构造函数 -> 对象字面量初始化”的统一流程; 其中某些写法只是该流程的简写形式。
- [必须] 若对象类型未显式声明任何构造函数,则视为存在默认的公开无参构造函数。
- [必须] `Type()`、`Type() { ... }`、`Type {}` 与 `Type { ... }` 都依赖当前作用域可见的无参构造; 若显式把无参构造声明为 `pr`,则在不可见作用域中不得使用这些写法。
- [必须] `Type(args)`、`Type(args) {}` 与 `Type(args) { ... }` 都依赖当前作用域可见且能匹配实参的构造函数。
- [必须] `let` 成员在成员声明及绑定初值、构造函数和对象字面量三个构造阶段内至多完成一次最终显式绑定。
- [必须] 若某个 `let` 成员在三个构造阶段都未完成显式绑定,则实例创建结束后保留其底层默认值。
- [必须] 三个构造阶段结束后,`let` 成员无论最终取显式绑定值还是默认值,都不得再绑定或赋值。
- [必须] 具体 `type` 进入 `spec` 位置前必须通过定义头或可见 `fit` 显式建立契约关系。
- [必须] `fit A { ... }` 形式的自扩展只为 `A` 补充成员,不单独建立新的 `spec` 契约关系。
- [必须] 若同一重载集合中的两个候选在当前可见的显式契约关系下可能同时匹配同一实参类型,则该重载集合非法,编译期报错。
- [必须] 对象字面量中每个成员按书写顺序逐个初始化; 同一字面量中同名成员最多只能出现一次,若重复出现则编译期报错。
- [必须] 对象字面量只能初始化当前作用域可访问的成员; 来自其他模块的 `pr` 成员不可在对象字面量中直接赋值。
- [必须] 构造函数中对 `self.<let成员>` 的直接赋值视为该成员在构造阶段完成最终显式绑定。
- [必须] 普通 `type` 不可直接出现在 `@fixed fn` 的参数或返回值位置,也不应成为 `@fixed type` 的直接依赖成员。
- [必须] 整数字面量赋值到具体整数类型时，若字面量值超出该类型的有效范围，编译期报错。
- [禁止] `void` 作为变量、成员、数组元素或普通值类型使用。
- [禁止] 在 `type` 内嵌套声明新类型。
- [禁止] 在成员类型位置内联声明匿名类型或新的具名类型。
- [建议] 对第三方或无法改动源码的类型,优先通过 `fit` 建立契约关系。

## 6 编译期

- 编译器必须检查 `type` 成员与实例创建路径的类型合法性与初始化路径。
- 编译器必须检查 `type` 声明位置是否满足模块级顶层约束。
- 编译器必须检查成员类型引用是否为已声明具名类型。
- 编译器必须检查成员 `let`/`var` 绑定是否合法,并按绑定规则验证赋值路径。
- 编译器必须按“成员声明及绑定初值 -> 构造函数 -> 对象字面量初始化”的顺序检查对象创建路径,并把所有对象构造写法归一到这一流程上,据此处理构造函数匹配、默认无参构造函数、构造可见性与对象字面量初始化。
- 编译器必须检查 `Type { ... }` 与 `Type {}` 是否正确解析为对应的无参构造加对象字面量初始化,并检查 `Type(args) { ... }` 与 `Type(args) {}` 是否正确解析为匹配构造加对象字面量初始化。
- 编译器必须检查对象创建阶段中 `let` 成员是否发生重复显式绑定。
- 编译器必须检查 `let` 成员在三个构造阶段结束后是否仍出现非法绑定或赋值。
- 编译器必须检查 `type` 与 `spec` 职责边界: 契约形状由 `spec` 定义,`type` 仅声明具体实现类型。
- 编译器必须检查标注 `@fixed` 的类型是否满足 ABI 约束。
- 编译器必须检查对象字面量中是否出现同名成员重复初始化,并在编译期报错。
- 编译器必须检查对象字面量中每个被初始化的成员在当前作用域是否可访问(`pr` 成员不可从外部字面量赋值)。
- 编译器必须检查类型重载与契约关系是否存在歧义或冲突,并在语义阶段报错阻止通过。
- 编译器必须检查整数字面量（含十六进制 `0x`、二进制 `0b`、八进制 `0o` 等各进制形式）是否超出目标整数类型的有效范围，超出时编译期报错。

## 7 运行时

- 无初始值绑定按类型规则得到默认零值。
- 对象类型采用托管引用语义; 赋值、传参与返回默认复制引用而非复制对象本体; 多个变量可同时引用同一对象实例,对任一引用的修改均可通过其他引用观察到。
- `string`、动态数组、对象类型与闭包环境由自动内存管理机制托管。
- 对象字面量阶段在构造函数执行完成后运行; `Type { ... }` 与 `Type {}` 分别等价于 `Type() { ... }` 与 `Type() {}`; `Type(args) { ... }` 与 `Type(args) {}` 则表示在匹配构造函数执行后再进入对象字面量初始化阶段。
- 因此对象创建的最终成员值取决于“成员声明及绑定初值 -> 构造函数 -> 对象字面量初始化”三阶段的有序合成结果。
- 标注 `@fixed` 的类型在 Feng 语言内部与普通 `type` 完全相同,采用托管引用语义: 赋值、传参与返回默认复制引用而非复制对象本体; 向 C ABI 边界传递时由编译器负责完成 ABI 兼容转换,开发者无需手动处理; 仅在 C ABI 边界受布局与类型资格约束。
- C 指针类型与 C 侧分配内存不受该机制管理。
- 对象类型上的 `==` / `!=` 默认比较引用身份; 两个不同实例即使成员值完全相同,仍不相等; 若需要值语义比较,应显式编写比较函数或成员方法。
- 整数运算在运行时发生溢出时，按对应位宽静默回绕，有符号与无符号类型均适用; 语言不提供运行时溢出检测或异常抛出。
