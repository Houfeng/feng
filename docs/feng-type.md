# Feng 语言类型规范

本文档用于补充 [feng-language.md](./feng-language.md) 中的类型系统概要说明,聚焦 feng 语言的数据类型、类型声明和类型约束细节。

## 1 类型系统概览

- Feng 采用强类型、静态类型系统,所有变量、参数和返回值均在编译期完成类型检查。
- 基础值类型覆盖有符号整数、无符号整数、浮点与布尔,并提供 `int` 和 `float` 作为常用别名。
- 统一使用 `type` 定义 feng 原生类型,使用 `extern type` 定义与 C 兼容的结构体、联合体或函数指针类型。
- 支持鸭子类型(鸭型),通过对象成员签名或函数签名完成编译期兼容性检查。
- 支持类型推导,但仅限带初始值的变量和可推导返回值的函数。
- 具名类型声明必须位于模块级顶层,不允许嵌套声明,不支持匿名类型定义,也不支持在类型位置直接内联声明类型; 同一模块内的 Feng 原生 `type` 与函数类型允许前向引用和互引用。

## 2 基础数据类型

### 2.1 数值与布尔类型

| feng 类型 | 别名 | 对应 C 类型 | 说明 | 默认零值 |
| --- | --- | --- | --- | --- |
| `i8` | - | `int8_t` | 8 位有符号整数 | `0` |
| `i16` | - | `int16_t` | 16 位有符号整数 | `0` |
| `i32` | - | `int32_t` | 32 位有符号整数 | `0` |
| `i64` | `int` | `int64_t` | 64 位有符号整数 | `0` |
| `u8` | - | `uint8_t` | 8 位无符号整数 | `0` |
| `u16` | - | `uint16_t` | 16 位无符号整数 | `0` |
| `u32` | - | `uint32_t` | 32 位无符号整数 | `0` |
| `u64` | - | `uint64_t` | 64 位无符号整数 | `0` |
| `f32` | - | `float` | 单精度浮点数 | `0.0` |
| `f64` | `float` | `double` | 双精度浮点数 | `0.0` |
| `bool` | - | `bool` | 布尔类型,值为 `true` / `false` | `false` |

### 2.2 字符串类型

- `string` 是 GC 托管字符串类型,默认零值为空字符串 `""`。
- `string` 不可用于 C 互操作。

### 2.3 数组类型

- 一维数组使用 `T[]` 语法,默认零值为空数组 `[]`。
- 多维数组沿用 `T[][]`、`T[][][]` 形式递进定义。
- 动态数组由 GC 托管,不可用于 C 互操作。

### 2.4 C互操作指针类型

- `*T` 对应 C 的 `T*`,默认零值为空指针。
- 指针类型仅用于 C 互操作。
- `extern type`、`*T` 指针和 C 侧分配的内存不受 GC 管理。

补充说明:

- `string`、动态数组和 feng 原生 `type` 对象由 GC 托管。

## 3 变量初始化概要

- 模块内变量和函数内局部变量必须显式使用 `let` 或 `var`; `let` 表示不可变绑定,`var` 表示可变绑定。
- 有初始值: 可省略类型,编译器自动推导,变量赋值为初始值。
- 无初始值: 必须显式标注类型,编译器自动赋予对应类型默认零值。
- `type` 与对象形式的 `extern type` 成员变量必须显式书写 `let` 或 `var`。
- `void` 不参与变量初始化规则,不可作为变量、成员、数组元素或普通值使用。
- 禁止使用未初始化的野值,全程保障类型安全。

`let` / `var` 的完整绑定规则、参数默认行为以及块作用域与名称屏蔽规则,见 [Feng 语言变量绑定与作用域规范](./feng-binding.md)。

```feng
// 有初始值,自动推导类型
let num = 10;
let str = "feng 语言";

// 无初始值,显式声明类型,自动赋零值
let age: int;
var p: Point; // C 兼容结构体自动零值初始化
```

## 4 Feng 原生自定义类型

统一使用 `type` 定义 feng 内部对象类型,支持成员变量、对象字面量、构造函数、成员方法和访问控制,不可与 C 直接互通。

规则说明:

- 成员变量必须显式添加 `var` 或 `let` 修饰。
- 支持 `Type { field: value }` 与 `Type {}` 形式的对象字面量; 未显式列出的成员按默认零值初始化。
- 若使用对象字面量创建对象,不会自动调用构造函数; 使用 `Type(args)` 时才会执行构造函数,且当前语言版本不会为 `type` 自动生成隐式默认构造函数。
- 同一模块内允许 Feng 原生 `type` 前向引用,因此不同 `type` 之间可以互引用。
- 成员类型必须引用已声明的具名类型,不能直接内联书写匿名对象结构。
- 类型成员默认公开,也可显式通过 `pu` 或 `pr` 设置可见性。
- 构造函数与类型同名,无返回值,无需 `new` 关键字即可实例化。
- 支持在 `type` 的成员 `fn` 与构造函数中使用 `self` 引用当前类型实例; 该绑定始终稳定,不会因别名赋值、参数传递或作为方法值使用而改变,而未定义在 `type` 中的普通 `fn` 或 `Lambda` 若直接使用 `self`,则编译期报错。

合法示例:

```feng
type A {
    var b: B;
}

type B {
    var a: A;
}
```

```feng
type User {
    var name: string;
    pr let id: int;
    var age: int;

    // 构造函数
    fn User(name: string, id: int, age: int) {
        self.name = name;
        self.id = id;
        self.age = age;
    }

    // 成员方法
    fn info() {
        return self.name;
    }
}
```

错误示例:

```feng
type User {
    var name: string;

    // 错误: 不允许在 type 内嵌套声明另一个普通类型
    type Address {
        var city: string;
    }
}
```

```feng
type A {
    // 错误: 不允许把成员类型直接写成匿名对象结构
    var c: {
        var c1: string;
        var c2: string;
    };
}
```

## 5 Feng 原生函数类型

使用 `type` 定义 feng 内部函数类型,仅用于 feng 内部,不可与 C 互通。函数类型声明必须位于模块级顶层,不支持在其他类型或函数类型签名中再次嵌套声明或内联声明新的函数类型。函数类型始终通过 `type 名称(参数): 返回值` 形式声明; `Lambda` 不是单独的函数类型,只是函数实现或函数字面量的一种写法,表达式语法中使用 `->`。

```feng
// feng 内部函数类型
type IntToInt(x: int): int;
type StringHandle(s: string): bool;
```

规则说明:

- 函数类型声明必须位于模块级顶层; 同一模块内允许前向引用,可在后续位置补充声明。
- 函数类型的参数类型和返回类型必须写成已有类型引用,不能在签名中继续内联声明另一个函数类型。
- `Lambda` 赋值、返回或传参时,其本质仍需满足目标函数类型签名。
- 函数类型同样参与鸭型兼容性检查,只要参数列表和返回值签名一致即可兼容。
- Feng 原生函数类型不可作为 C 函数指针传递。
- `extern type` 定义的函数类型仅用于 C ABI 兼容回调。

错误示例:

```feng
type User {
    var name: string;

    // 错误: 不允许在普通类型内嵌套声明函数类型
    type NameGetter(): string;
}
```

```feng
// 错误: 不允许在函数类型签名中内联声明另一个函数类型
type FormatName(p1: string): (p2: string): string;
```

## 6 鸭子类型(鸭型)规则

- 遵循“只要像鸭子一样行走、鸣叫,就是鸭子”的设计逻辑。
- 类型无需显式实现接口或继承,对象类型只要拥有相同名称和签名的成员方法即可兼容,函数类型只要参数列表和返回值签名一致即可兼容。
- 编译期完成鸭型兼容性检查,保障运行期安全。

对象类型示例:

```feng
type Duck {
    fn quack(): string {
        return "quack";
    }
}

type Bird {
    fn quack(): string {
        return "bird quack";
    }
}

// 兼容所有包含 quack() 方法的类型
fn make_sound(obj: Duck) {
    print(obj.quack());
}
```

函数类型示例:

```feng
type IntHandler(x: int): int;
type NumberMapper(v: int): int;

fn apply(handler: IntHandler, value: int): int {
    return handler(value);
}

fn run(mapper: NumberMapper, value: int): int {
    // NumberMapper 与 IntHandler 签名一致,因此可直接兼容
    return apply(mapper, value);
}
```

## 7 类型相关核心约束

- 强类型、静态类型,无隐式类型转换。
- 无 `any` 类型,未初始化变量自动赋零值。
- 变量有初始值可自动推导类型。
- 参数必须显式标注类型,默认等价于 `let`,使用 `var` 可声明可变参数。
- `void` 代表空无类型,无返回值函数可省略 `void`,有返回值时可自动推导。
- 具名类型声明必须位于模块级顶层; 不允许嵌套声明,不支持匿名类型定义,也不支持在类型位置直接内联声明类型; 同一模块内的 Feng 原生 `type` 与函数类型允许前向引用和互引用。
- 数组使用 `T[]`,支持多维数组,动态数组由 GC 托管。
- 函数类型定义统一使用 `type ... : 返回值` 形式,`->` 仅用于 `Lambda` 表达式语法,不引入新的函数类型。
- 鸭型规则同时适用于对象类型和函数类型。
- 无冗余关键字,无结构体专用关键字,统一 `type` 体系。
- 成员变量必须使用 `var` 或 `let` 修饰; 类型成员默认公开,也可显式通过 `pu` 和 `pr` 设置可见性。
