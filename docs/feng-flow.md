# Feng 语言流程控制规范

本文档用于补充 [feng-language.md](./feng-language.md) 中的流程控制概要说明,聚焦 Feng 语言的条件判断、`if` 表达式、循环与控制转移规则。

## 1 流程控制概览

- Feng 提供条件分支、`if` 表达式、循环和结构化异常处理。
- 条件判断支持 `if / else if / else` 形式。
- `if` 支持作为表达式使用,用于条件分支求值。
- 循环支持 `while`、`for`（三段式）、`for/in`（迭代）、`break` 和 `continue`。
- `try / catch / finally` 与控制流的详细异常语义见 [Feng 语言异常模型规范](./feng-exception.md)。

## 2 普通条件判断

规则说明:

- `if` 和 `else if` 的条件表达式必须是 `bool`。
- 条件分支按从上到下顺序依次判断,命中第一个为 `true` 的分支后即停止后续判断。
- `else` 分支可省略，省略时表示条件全部不满足则直接离开整个分支结构。

```feng
if a > b {
  // 逻辑代码
} else if a == b {
  // 逻辑代码
} else {
  // 逻辑代码
}
```

## 3 条件匹配形式

Feng 额外支持以 `if 目标值 { ... }` 形式书写的条件匹配表达式,其本质是按顺序执行的相等性分支匹配。

规则说明:

- 匹配目标表达式先求值一次。
- 匹配目标类型仅支持 `所有整型`、`string`、`bool`。
- 每个分支的标签部分支持以下三种形式，可在同一分支中混合使用，以逗号分隔：
  - **单值**：字面量，或初始化器为字面量的 `let` 绑定；不允许运行时表达式、含运算符的常量折叠表达式或跨绑定传播。
  - **值列表**：多个单值以逗号分隔，如 `10, 19, 29`；支持所有条件匹配允许的类型。
  - **整数闭区间**：`m...n` 表示 `[m, n]`（两端均包含），仅支持整型；`m`、`n` 须为单值形式的编译期常量，且 `m <= n`，否则编译期报错。
  - 同一分支内可混合以上形式，如 `0, 10...18, 100`。
- 分支按书写顺序匹配，命中第一个覆盖目标值的分支后返回对应结果。
- 编译器对所有分支标签进行交叉检测：单值重复、值列表元素与其他分支重叠、区间与单值重叠、区间与区间重叠，均视为不可达死代码，编译期报错。
- `else` 分支可省略，省略时表示条件全部不满足则直接离开整个分支结构。

```feng
if age {
  0 { 
    // 逻辑代码
  }
  1...17 { 
    // 逻辑代码
  }
  18, 20, 22 { 
    // 逻辑代码
  }
  19, 23...59 { 
    // 逻辑代码
  }
  else { 
    // 逻辑代码
  }
}
```

## 4 `if` 表达式

`if` 可作为表达式参与赋值、返回值计算或参数传递。表达式形式的 `if` 会返回被选中分支的值。

规则说明:

- `if` 表达式，支持上述 2 种 `if` 形式。
- `if` 表达式，必须有返回值，最后一行作为返回值，且返回值类型必须一致，且不可省略 `else` 分支。
- 各分支块中，块值表达式末尾的分号可加可不加，不影响块值提取；编译器始终取最后一个表达式的值作为该分支的结果。

```feng
let label = if age >= 18 {
  "adult";
} else {
  "minor";
}
```

```feng
let label = if age {
  0 { "婴儿"; }
  18 { "成年"; }
  60 { "老年"; }
  else { "青年"; }
}
```

## 5 `while` 循环

- `while` 条件表达式必须为 `bool`。
- 每次进入循环体前都会重新计算条件。
- 条件为 `false` 时立即退出循环。

```feng
var i = 0;
while i < 3 {
  print(i);
  i = i + 1;
}
```

## 6 `for` 循环

`for` 支持两种形式：三段式循环与 `for/in` 迭代循环。

### 6.1 三段式 `for`

```feng
for 初始化; 条件; 更新 {
  // 循环体
}
```

规则说明：

- 初始化子句可为空，也可使用局部绑定或赋值语句。
- 条件子句可为空；为空时表示恒为 `true`。
- 更新子句可为空，通常写赋值或函数调用。
- 初始化子句中声明的绑定仅在当前 `for` 循环内部可见。
- 初始化子句中的绑定必须显式写 `let` 或 `var`，不可省略。
- 若更新子句对该绑定赋值，则必须声明为 `var`，否则编译期报错。

```feng
for var i = 0; i < 10; i = i + 1 {
  print(i);
}
```

### 6.2 `for/in` 循环

```feng
for let it in items {
  // 循环体
}
```

规则说明：

- 循环变量必须显式写 `let` 或 `var`，不可省略。
- `let` 绑定在循环体内不可重新赋值，`var` 绑定可在循环体内修改，但不影响序列本身及下一轮的绑定值。
- 循环变量仅在循环体内可见；每轮迭代产生独立的新绑定。
- 循环序列暂时仅支持数组 `T[]`。

## 7 `break` 与 `continue`

- `break` 仅可用于 `while` 或 `for` 循环内部,用于立即退出当前最近的一层循环。
- `continue` 仅可用于循环内部,用于跳过当前轮剩余语句并进入下一轮判断。
- `break` 和 `continue` 都不携带返回值。

## 8 与异常模型的关系

- `try / catch / finally` 本身属于控制流结构的一部分。
- `finally` 会在 `return`、`break`、`continue` 和异常展开前执行。
- `throw`、异常传播和 ABI 边界限制的详细规则见 [Feng 语言异常模型规范](./feng-exception.md)。
