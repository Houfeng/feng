# Feng 语言对象模型规范

本文档用于补充 [feng-language.md](./feng-language.md) 中的对象模型概要说明,聚焦 Feng 原生 `type` 的实例创建、引用语义、成员可变性与方法调用规则。

> **设计范围说明**: 本文档讨论普通 Feng `type` 的对象模型。`@fixed type` 属于非托管 ABI 值类型,其布局与 ABI 规则见 [feng-interop.md](./feng-interop.md)。

## 1 对象模型概览

- Feng 原生 `type` 表示 GC 托管的引用对象类型。
- `@fixed type` 表示 C 兼容 ABI 值类型,不属于 Feng 原生对象模型核心。
- `type` 实例在赋值、传参和返回时默认复制引用,而不是复制整个对象。
- `let` / `var` 作用于绑定和成员可变性,不等价于深度不可变对象。
- 对 Feng 原生 `type` 而言,实例创建可包含最多三个有序阶段: 成员声明初值、构造函数、对象字面量初始化。

## 2 `type` 实例的创建方式

Feng 原生对象支持三种创建语法:

- 仅调用构造函数: `Type(args)`
- 调用无参构造后执行对象字面量初始化: `Type { field: value }` 或 `Type() { field: value }`
- 调用匹配构造函数后执行对象字面量初始化: `Type(args) { field: value }`

### 2.1 对象字面量初始化

对象字面量初始化支持 `Type { field: value }`、`Type() { field: value }`、`Type(args) { field: value }` 三种形式,也允许使用空对象字面量 `Type {}` 与 `Type() {}`。

规则说明:

- 对象字面量初始化总是建立在某个构造阶段之后执行; `Type { ... }` 是 `Type() { ... }` 的简写形式。
- 成员声明中带初始值的成员先完成第一阶段显式绑定; 未声明初值的成员底层存储先放入类型默认值,但这不视为 `let` 已完成显式绑定。
- 先执行选定的构造函数,再执行对象字面量初始化。
- 字面量中显式列出的成员按名称逐个初始化,执行顺序与书写顺序一致。
- 同一个对象字面量中,同名成员最多只能出现一次; 若重复列出同一成员名,编译期报错。
- 对象字面量只能初始化当前作用域可访问的成员; 来自其他模块的 `pr` 成员不可在对象字面量中直接赋值。
- `let` 成员在整个创建流程中只能完成一次最终显式绑定: 若已在成员声明初值或构造函数阶段绑定,则对象字面量中不得再次绑定; 若前两阶段未绑定,则可在对象字面量阶段完成绑定。
- 构造函数中对 `self.<let成员>` 的直接赋值,视为该成员在构造阶段完成最终显式绑定。
- `var` 成员不受单次最终绑定限制。
- `Type {}` 与 `Type() {}` 表示在无参构造完成后不再追加任何字面量初始化。

```feng
let user = User { name: "guest", age: 18 };
let empty = User {};
let user2 = User("guest") { age: 18 };
```

### 2.2 构造函数调用

`Type(args)` 表示调用与类型同名的构造函数。若一个 `type` 未显式声明任何构造函数,则视为存在一个默认的公开无参构造函数,因此 `Type()`、`Type {}` 与 `Type() {}` 都合法。若该 `type` 已显式声明了一个或多个构造函数,则只按“名称 + 参数个数 + 参数类型”参与重载决议,并且是否可写 `Type()`、`Type {}` 或 `Type() {}` 取决于是否存在可访问的无参构造函数。

```feng
type User {
    var name: string;
    pr let id: int;
    var age: int;

    fn User(name: string, id: int, age: int) {
        self.name = name;
        self.id = id;
        self.age = age;
    }
}

let user = User("guest", 1, 18);
```

补充规则:

- 若 `type` 未显式声明任何构造函数,编译器视为其存在默认的公开无参构造函数。
- `Type {}` 是 `Type() {}` 的简写,两者都要求存在可访问的无参构造函数。
- `Type(args) { field: value }` 先做构造函数重载决议,再执行对象字面量初始化; 若不存在匹配且可访问的构造函数,则编译期报错。
- 想创建所有成员都经由无参构造建立且不追加字面量初始化的实例时,可使用 `Type()`、`Type {}` 或 `Type() {}`。
- 想创建带部分初始值的实例时,可使用 `Type { field: value }` 或 `Type(args) { field: value }`。
- 若 `let` 成员已在成员声明初值或构造函数中完成显式绑定,则对象字面量中不得再次为其赋值; 其中构造函数中对 `self.<let成员>` 的直接赋值属于显式绑定。

示例:

```feng
let t1 = Type {};
let t2 = Type() {};
let t3 = Type(x) { foo: 123 };
let t4 = Type(x);
```

- 上述四种写法在语法上都合法。
- `t1` 与 `t2` 是否编译通过,取决于 `Type` 是否存在可访问的无参构造函数; 若未显式声明任何构造函数,则使用默认的公开无参构造函数。
- `t3` 是否编译通过,取决于是否存在与参数 `x` 匹配且可访问的构造函数,以及后续对象字面量是否违反 `let` 的单次最终绑定规则。
- `t4` 在语法上同样合法,表示仅执行与参数 `x` 匹配的构造函数而不追加对象字面量初始化; 是否编译通过,取决于是否存在匹配且可访问的构造函数。

## 3 赋值、传参与返回语义

Feng 原生 `type` 实例采用引用语义。

规则说明:

- 变量赋值只复制对象引用,不复制对象本体。
- 函数参数传递和返回值传递同样只复制引用。
- 多个变量可同时引用同一个对象实例。
- 若需要独立对象,必须显式重新构造。

```feng
let a = User { name: "guest" };
let b = a;
b.name = "admin";
```

上例中,`a` 与 `b` 指向同一个对象,因此对 `b.name` 的修改也会通过 `a` 观察到。

## 4 绑定不可变与成员可变性

`let` / `var` 在对象场景中分为两层语义:

- 绑定层: 变量名是否允许重新绑定到另一个对象
- 成员层: 对象的具体成员是否允许被改写

规则说明:

- `let user = ...;` 表示变量 `user` 不可重新绑定,但不自动冻结对象。
- 若对象成员声明为 `var`,即使对象存放在 `let` 绑定中,仍可修改该成员。
- 若对象成员声明为 `let`,则该成员在初始化后不可再次赋值。

```feng
type User {
    var name: string;
    let created_at: int;
}

let user = User { name: "guest", created_at: 1 };
user.name = "admin";
```

## 5 方法与 `self`

- 成员方法通过 `obj.method(args)` 调用。
- 编译器在成员方法中隐式提供 `self`,指向当前对象实例。
- 构造函数中的 `self` 与成员方法中的 `self` 指向当前正在初始化或调用的对象。
- `self` 只在定义于 `type` 中的成员 `fn` 或构造函数内合法; 无论该 `fn` 是直接调用还是以 `obj.method` 形式作为函数值传递,`self` 都始终稳定绑定到对应的类型实例,而未定义在 `type` 中的普通 `fn` 或 `Lambda` 若直接使用 `self`,则编译期报错。

```feng
type Counter {
    var value: int;

    fn inc() {
        self.value = self.value + 1;
    }
}
```

```feng
type User {
    fn say() {}
    fn say(msg: string) {}
}

type M0(): void;

fn test(m: M0) {
    m();
}

let u = User {};
test(u.say);
let s_ok: M0 = u.say;
let s = u.say; // 错误: say 存在多个重载,当前上下文无法推导 s 的函数类型
```

上例中,`u.say` 在作为参数传给 `test` 时,传递的是一个已绑定到 `u` 所引用实例的方法值; 由于目标类型 `M0` 能唯一确定无参重载,`test` 内部执行 `m()` 时,`say` 中的 `self` 仍然指向该 `User` 实例,不会因为被当作普通函数值传递而改变。`let s_ok: M0 = u.say;` 同样合法,因为显式声明的目标函数类型足以唯一选定无参重载。`let s = u.say;` 则因为缺少足以唯一选定重载的目标函数类型,必须在编译期报错。

## 6 对象身份与相等性

- Feng 原生 `type` 对象上的 `==` / `!=` 默认比较引用身份。
- 两个不同实例即使成员值完全相同,仍视为不相等。
- 若需要值语义比较,应显式编写比较函数或成员方法。

## 7 与 `@fixed type` 的边界

- 普通 `type` 是 Feng 原生对象,受运行时自动内存管理机制管理。
- `@fixed type` 与 `@fixed @union type` 是非托管 ABI 值类型,不受运行时 RC 或 GC 接管。
- 普通 `type` 不可直接出现在 `@fixed fn` 的参数或返回值中,也不应成为 `@fixed type` 的直接依赖成员。
- `@fixed type` 的方法不改变实例布局; 其 ABI 资格由方法自身是否标记为 `@fixed` 决定。
