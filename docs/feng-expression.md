# Feng 语言表达式与运算规范

本文档用于补充 [feng-language.md](./feng-language.md) 中的表达式概要说明,聚焦 Feng 语言中的表达式类别、运算符优先级、求值顺序与赋值规则。

## 1 表达式系统概览

- Feng 的表达式用于计算值,其结果始终在编译期具有确定类型。
- 表达式求值遵循强类型规则,禁止隐式类型转换。
- 数值运算在同类型操作数之间直接执行; 跨数值类型运算前必须显式转换。
- `=` 属于赋值语句,不参与普通表达式优先级计算。
- `if` 表达式属于流程控制的一部分,其详细规则见 [Feng 语言流程控制规范](./feng-flow.md)。

## 2 表达式类别

Feng 支持以下常见表达式形式:

- 字面量: 整数、浮点、`bool`、`string`
- 名称引用: 变量、参数、模块成员、类型成员
- 数组字面量: `[]`、`[1, 2, 3]`
- 对象字面量: `Type { field: value }`
- 函数调用: `add(a, b)`
- 成员访问: `obj.name`
- 下标访问: `arr[index]`
- 一元运算: `-x`（数值取负）、`!x`（逻辑非，仅适用于 `bool`）、`~x`（按位非，仅适用于整数）
- 二元运算: `+ - * / %`、比较运算、逻辑运算、位运算（`&`、`|`、`^`、`<<`、`>>`）
- 显式类型转换: `(目标类型)表达式`

示例:

```feng
let total = add(1, 2) * 3;
let ok = total > 0 && total < 10;
let item = users[0];
let name = item.name;
```

## 3 运算语义与类型约束

- 算术运算仅适用于数值类型,左右两侧类型必须一致或能在编译期通过显式转换统一。
- 比较运算返回 `bool`,左右两侧必须是可比较的同类值。
- 逻辑运算仅适用于 `bool`。
- 位运算（`&`、`|`、`^`、`<<`、`>>`）及按位非（`~`）仅适用于整数类型，两侧操作数类型必须相同，不支持隐式提升。
- 字符串可使用 `+` 进行拼接。
- `type` 对象、对象形式的 `@fixed type` 和动态数组上的 `==` / `!=` 默认比较引用身份,不执行深度比较。
- 基础数值、`bool` 与 `string` 的 `==` / `!=` 按值比较; `string` 的值语义指内容相等,不依赖底层引用身份。

### 3.1 数值运算规则

Feng 的数值运算参考 C 的常见数值行为,但保留强类型约束: 不做隐式数值提升,只有在两侧类型一致时才可直接运算。

规则说明:

- `+`、`-`、`*` 适用于整数与浮点类型。
- `/` 适用于整数与浮点类型。
- `%` 适用于整数与浮点类型。
- 不同数值类型之间不能直接运算,例如 `i32 + i64`、`int + float` 都必须先显式转换。
- 整数除法参考 C,结果向零截断。
- 整数取模参考 C,余数符号与左操作数一致。
- 浮点取模使用余数语义: 结果等于 `a - trunc(a / b) * b`,其中 `trunc` 表示向零截断; 结果类型与操作数类型一致,结果符号与左操作数一致。
- 浮点运算结果沿用目标平台对应 C 浮点实现。
- 除零在整数运算中属于未定义行为,语言层禁止依赖该行为; 在浮点运算中沿用目标平台对应 C 浮点实现; 对 `%` 而言,无论整数还是浮点,右操作数为 `0` 时都属于未定义行为。
- 整数运算溢出按对应位宽静默回绕，有符号与无符号均适用; 语言不提供运行时溢出检测。
- 编译期可确定的整数溢出（如字面量赋值超出目标类型范围）在编译期报错，不等待运行时触发。

```feng
let a: i32 = 7;
let b: i32 = 3;
let q = a / b; // 结果为 2
let r = a % b; // 结果为 1

let x: float = 7.8;
let y: float = 3.2;
let rem = x % y; // 结果约为 1.4
```

### 3.2 逻辑运算规则

逻辑运算仅适用于 `bool` 类型，不适用于整数、浮点、`string` 或对象类型。

| 符号 | 名称 | 说明 |
| --- | --- | --- |
| `&&` | 逻辑与 | 两侧均为 `true` 时结果为 `true`，否则为 `false` |
| `\|\|` | 逻辑或 | 两侧只要有一个为 `true` 则结果为 `true`，否则为 `false` |
| `!` | 逻辑非 | 对 `bool` 值取反；`true` 变 `false`，`false` 变 `true` |

规则说明:

- `&&` 与 `\|\|` 均使用短路求值：`&&` 当左侧为 `false` 时不求值右侧；`\|\|` 当左侧为 `true` 时不求值右侧。
- `!` 是一元前缀运算符，仅作用于 `bool` 类型，不可用于整数（按位非请使用 `~`）。
- 逻辑运算的操作数与结果类型均为 `bool`，不存在隐式的数值到 `bool` 转换。
- `&&` 优先级高于 `\|\|`，两者均低于比较运算符（`<`、`<=`、`>`、`>=`、`==`、`!=`）。

```feng
let a = true;
let b = false;
let and_val = a && b;   // false
let or_val  = a || b;   // true
let not_val = !a;       // false

let x: int = 5;
let in_range = x > 0 && x < 10;   // true，短路：x > 0 先求值
```

### 3.3 位运算规则

位运算仅适用于整数类型（有符号与无符号），不适用于浮点、`bool`、`string` 或对象类型。

| 符号 | 名称 | 说明 |
| --- | --- | --- |
| `&` | 按位与 | 对应位均为 1 则为 1，否则为 0 |
| `\|` | 按位或 | 对应位只要有 1 则为 1，否则为 0 |
| `^` | 按位异或 | 对应位不同则为 1，相同则为 0 |
| `~` | 按位非 | 所有位取反 |
| `<<` | 左移 | 向左移动指定位数，右侧补 0 |
| `>>` | 右移 | 向右移动指定位数；有符号整数算术右移（补符号位），无符号整数逻辑右移（补 0） |

规则说明:

- 位运算的两侧操作数类型必须是相同整数类型，不支持隐式类型提升。
- `~` 仅作用于整数类型，对操作数每一位取反；`!` 仅作用于 `bool` 类型，表示逻辑非；两者是独立运算符，不可混用。
- `<<` 与 `>>` 的右操作数为移位位数，必须为非负整数；移位量超出类型位宽时为未定义行为，语言层禁止依赖该行为；编译期可确定的越界移位量（如字面量）在编译期报错，运行时动态值越界行为不做保护。
- 有符号整数右移（`>>`）按算术右移处理，高位补符号位。
- 无符号整数右移（`>>`）按逻辑右移处理，高位补 0。
- 左移（`<<`）不区分有符号与无符号，统一右侧补 0；移出的高位直接丢弃。

```feng
let a: i32 = 10;    // 二进制 0b0000_1010
let b: i32 = 12;    // 二进制 0b0000_1100
let and_val = a & b;    // 8  (0b0000_1000)
let or_val  = a | b;    // 14 (0b0000_1110)
let xor_val = a ^ b;    // 6  (0b0000_0110)
let not_val = ~a;       // 按位取反，每一位翻转
let shl = a << 1;       // 20 (0b0001_0100)
let shr = b >> 1;       // 6  (0b0000_0110)
```

### 3.4 显式类型转换

Feng 采用与 C 相同风格的显式转换语法:

```feng
(目标类型)表达式
```

规则说明:

- 显式转换只改变结果类型,不会改变原始表达式绑定的静态类型。
- 数值类型之间的转换必须显式写出,编译器不会自动插入转换。
- 整数到更小位宽整数的转换会按目标位宽截断高位。
- 浮点到整数的转换参考 C,小数部分直接截断。
- 整数到浮点的转换在目标浮点类型可表示范围内按值转换; 超出精确表示范围时允许产生精度损失。
- `bool` 不参与数值显式转换,不能将数值直接转换为 `bool`,也不能将 `bool` 转换为数值类型。
- `string`、数组、`type`、`@fixed type` 与数值类型之间不支持数值转换。

```feng
let total: i64 = 100;
let small = (i32)total;
let ratio = (float)small / 3.0;
let size: u8 = (u8)257;
```

上例中,`(u8)257` 的结果按目标位宽截断。

## 4 求值顺序

Feng 为可预测性采用从左到右的求值顺序。

规则说明:

- 二元运算从左操作数开始求值,再求值右操作数。
- 函数调用先求值被调用目标,再按从左到右顺序求值每个参数。
- 成员访问、下标访问和调用属于后缀运算,按从左到右连续展开。
- `&&` 和 `||` 使用短路求值。
- `if` 表达式只求值被选中的分支。

```feng
let value = get_user().scores[index + 1];
```

在上例中,先求值 `get_user()`、再求值 `index + 1`,最后执行下标访问。

## 5 运算符优先级

运算符按以下优先级从高到低结合:

| 优先级 | 形式 | 说明 | 结合方向 |
| --- | --- | --- | --- |
| 1 | `expr(...)` `expr.member` `expr[index]` | 调用、成员访问、下标访问 | 从左到右 |
| 2 | `~x` `-x` `!x` | 按位非（整数）、数值取负、逻辑非（`bool`） | 从右到左 |
| 3 | `*` `/` `%` | 乘除模 | 从左到右 |
| 4 | `+` `-` | 加减 | 从左到右 |
| 5 | `<<` `>>` | 左移、右移 | 从左到右 |
| 6 | `<` `<=` `>` `>=` | 比较运算 | 从左到右 |
| 7 | `==` `!=` | 相等性比较 | 从左到右 |
| 8 | `&` | 按位与 | 从左到右 |
| 9 | `^` | 按位异或 | 从左到右 |
| 10 | `\|` | 按位或 | 从左到右 |
| 11 | `&&` | 逻辑与 | 从左到右 |
| 12 | `\|\|` | 逻辑或 | 从左到右 |

补充规则:

- 不支持 `++`、`--`、逗号表达式等 C 风格副作用运算符。
- 复杂表达式可使用 `()` 显式改变求值顺序。

## 6 字面量与聚合构造

### 6.1 数值字面量

整数字面量的默认类型为 `int`（即 `i32`），浮点字面量（含小数点或科学计数法指数）的默认类型为 `double`（即 `f64`）。语言不提供 `L`、`F`、`U` 等后缀语法；需要指定其他数值类型时，必须使用显式类型声明。

整数字面量支持以下进制前缀:

| 前缀 | 进制 | 示例 |
| --- | --- | --- |
| 无前缀 | 十进制 | `255` |
| `0x` | 十六进制 | `0xFF` |
| `0b` | 二进制 | `0b1111_1111` |
| `0o` | 八进制 | `0o377` |

规则说明:

- 不含小数点且不含指数的数字字面量始终为整数类型，默认 `int`；含小数点或科学计数法指数的数字字面量为浮点类型，默认 `double`。
- 需要其他具体数值类型（如 `i32`、`u8`、`f32`）时，必须通过显式类型声明指定，不支持后缀写法。
- 字面量中可用 `_` 作为数字分隔符以提高可读性；`_` 不得出现在字面量开头、结尾，以及进制前缀（`0x`/`0b`/`0o`）紧邻之后。
- 十六进制字母大小写均可（`0xAB` 与 `0xab` 等价）。
- 浮点字面量支持科学计数法（`1.5e3`、`2.0e-4`），指数部分同样以默认 `double` 类型处理。

```feng
let a = 255;           // int，十进制
let b = 0xFF;          // int，十六进制
let c = 0b1111_1111;   // int，二进制
let d = 0o377;         // int，八进制
let e = 3.14;          // double，默认浮点类型
let f = 1.5e3;         // double，科学计数法，值为 1500.0
let big = 1_000_000;   // int，使用 _ 分隔提高可读性

let x: i32 = 255;      // 显式指定为 i32
let y: u8  = 0xFF;     // 显式指定为 u8
let z: f32 = 3.14;     // 显式指定为 f32
```

### 6.2 数组字面量

- 数组字面量使用 `[]` 包裹。
- 同一数组字面量中的元素类型必须一致。
- 空数组字面量 `[]` 只能在目标类型可确定时使用。

```feng
let nums = [1, 2, 3];
let matrix = [[1, 2], [3, 4]];
```

### 6.3 对象字面量

对象字面量支持 `Type { field: value }`、`Type {}`、`Type() { field: value }` 与 `Type(args) { field: value }` 等形式,适用于 `type` 与对象形式的 `@fixed type`。字段按名称书写,顺序不限,同名字段不得重复出现,当前作用域不可访问的 `pr` 字段不可在字面量中出现。对象字面量的创建语义与构造阶段规则见 [Feng 语言类型规范](./feng-type.md)。

```feng
let user = User { name: "guest", age: 18 };
let point = Point { x: 10, y: 20 };
```

## 7 赋值与可变性规则

`=` 是赋值语句,不产生表达式值。

规则说明:

- 赋值左侧必须是可写位置: `var` 变量、`var` 成员或可写下标位置。
- `let` 绑定和 `let` 成员在初始化完成后不可再次赋值。
- 赋值先完整求值右侧表达式,再写入左侧目标。
- 赋值可出现在普通语句位置以及 `for` 的初始化/更新子句中。

```feng
var count = 0;
count = count + 1;

let user = User { name: "guest" };
user.name = "admin";
```

上例中,`user` 是 `let` 绑定,表示该变量名不可重新绑定到其他对象; 但若 `name` 是 `var` 成员,仍允许修改成员值。
