# Feng 语言核心规范

> **设计原则基础**: 本文档的摘要规则建立在 [Feng 语言设计原则](./feng-principles.md) 之上。
> 尤其是: 注解只影响语义分析与代码生成,不改变语法; 只有关键字能够影响语法类别,语法问题应能在 parse 阶段检查出来。

## 1 设计哲学

Feng 是一门**强类型、静态类型、支持 `spec` 契约与 `fit` 显式适配**的编程语言,秉持极简主义设计理念,语法紧凑、关键字稀少,兼顾脚本语言的简洁性与 C 语言的高效性; 支持闭包、函数式编程与结构化异常处理,全程保障内存安全; 以 `type` 统一定义具名类型,并通过 `extern fn`、`@fixed` 和调用方式注解实现与 C 语言的安全互操作,底层可无缝映射为 C 语言代码; 普通 Feng 对象由运行时自动管理,实现上可采用 RC、GC 或等价策略,`@fixed` 边界上的 ABI 值不受该机制接管; 同时支持统一的 `.fb` 二进制包格式分发,可携带自有 ABI 静态库层与 C ABI 兼容层,兼顾源码保护、编译加速与跨语言复用。

## 2 语言核心特性

- 静态类型: 变量、参数、返回值类型在编译期确定,运行期不可更改。
- 强类型: 严格类型检查,禁止隐式类型转换,类型不匹配直接编译报错。
- 无 `any` 类型: 不提供动态任意类型,所有变量、参数均为明确类型。
- 显式契约: `spec` 用于声明契约形状,具体 `type` 进入 `spec` 位置必须通过定义头或可见的 `fit` 显式建立关系,同包与跨包都不自动按结构兼容。
- 支持闭包: `Lambda` 表达式可捕获外部作用域变量,延长变量生命周期。
- 类型推导: 变量有初始值时,可省略类型标注,编译器自动推导; 但若初始值是重载函数或重载方法值,且当前上下文无法唯一确定目标函数类型,则编译期报错。
- 表达式规则: 提供确定的运算符优先级、从左到右求值顺序、参考 C 的数值运算规则与无隐式转换语义。
- 函数返回值: `void` 代表空无类型,无返回值时可省略,有返回值可自动推导类型。
- 绑定规则: `let` 用于声明不可变绑定,`var` 用于声明可变绑定; 参数可省略 `let` / `var`,省略时默认按 `let` 处理,模块内变量、函数内变量和类型成员必须显式书写 `let` 或 `var`。
- 类型使用规范: 具名类型声明必须位于模块级顶层,不允许嵌套声明,不支持匿名类型定义,也不支持在类型位置直接内联声明类型; 同一模块内的 Feng 原生 `type` 与函数类型允许前向引用和互引用。
- 数组支持: 采用 `T[]` 语法,支持一维及多维数组,如 `T[][]`、`T[][][]`。
- 异常处理: 支持 `throw` 与完整 `try/catch/finally` 结构化异常处理,异常不得穿越 C ABI 边界。
- 默认零值: 未初始化变量由编译器自动赋予对应类型默认零值。
- 对象模型: Feng 原生 `type` 采用托管引用语义,对象字面量与构造函数并存。
- 内存管理: 普通 `type`、`string`、动态数组和闭包等托管对象由运行时自动管理,实现上可采用 RC、GC 或等价策略; `@fixed` 仅额外约束进入 C ABI 边界时的布局与资格,不单独改变 Feng 侧的默认零值、比较、绑定或创建语义。
- C 互操作核心: `extern fn` 仅用于声明 C 外部函数; `@fixed type` 用于定义 C 兼容结构体、联合体和函数指针类型; `@fixed fn` 用于定义 Feng 实现的 ABI 兼容回调或导出函数; 对无函数体的 `extern fn` 导入声明,使用带参数的 `@cdecl("libname_or_path")`、`@stdcall("libname_or_path")` 或 `@fastcall("libname_or_path")` 指定原生库来源与调用方式,全程进行编译期 ABI 与类型安全检查。
- 包分发特性: 支持统一的 `.fb` 包格式,可选携带自有 ABI 静态库层与 C ABI 兼容动态库层,支持闭源复用与编译加速。

## 3 基础语法规则

1. 文件与包扩展名:
    - `.ff`: 源文件（Feng File）
    - `.fm`: 项目或包清单文件（Feng Manifest）
    - `.fi`: 编译后包的公开接口及元信息文件（Feng Interface）
    - `.fb`: 包文件（Feng Bundle）
2. 语句结束符: 分号 `;`
3. 代码块: 统一使用花括号 `{}` 包裹
4. 注释规范:
   - 单行注释: `// 注释内容`
   - 多行注释: `/* 注释内容 */`
5. 代码书写顺序: 模块声明 → 模块导入 → 类型定义 → 函数/业务代码,不可颠倒

## 4 关键字、保留字与内建注解

当前规范共定义 `23` 个关键字、`12` 个保留字和 `6` 个内建注解。

规则说明:

- 关键字: 当前版本已经参与语法,词法阶段会直接识别为关键字 token,不得作为标识符。
- 保留字: 当前版本尚未赋予对应语法能力,但为语言演进预留,当前同样不得作为标识符。
- 内建注解: 以 `@` 开头,用于补充语义分析与代码生成信息,不属于普通标识符。
- 例如: `self` 是关键字,`this` 是保留字。

### 4.3 所有内建注解（共6个）

下表按当前全部规范文档汇总 Feng 的内建注解。

| 内建注解 | 适用位置 | 含义 |
| --- | --- | --- |
| `@fixed` | `type`、`fn` 和方法声明前 | 表达“该声明希望进入 ABI 固定边界”的语义资格。对象形式的 `@fixed type` 可声明 C 兼容结构体或联合体; 函数形式的 `@fixed type` 可声明 C 兼容函数指针类型; `@fixed fn` 或 `@fixed` 方法可声明 Feng 实现的 ABI 兼容函数。`@fixed` 不改变语法形式,其合法性由语义分析检查。 |
| `@union` | 对象形式的 `@fixed type` 声明前 | 将对象形式的 `@fixed type` 声明为 C 联合体; 未标注时默认按 C 结构体处理。 |
| `@cdecl` | 无函数体的 `extern fn` 声明前,或 `@fixed fn` / `@fixed` 方法定义前 | 带一个参数时写作 `@cdecl("libname_or_path")`,表示该无函数体 `extern fn` 从指定原生库导入且使用 `cdecl`; 该参数也可引用当前 `mod` 作用域中以字符串字面量初始化的 `let` 绑定。无参数时用于显式声明 `@fixed fn` 或 `@fixed` 方法的 C ABI 调用方式; 未显式标注时默认按 `cdecl` 处理。 |
| `@stdcall` | 无函数体的 `extern fn` 声明前,或 `@fixed fn` / `@fixed` 方法定义前 | 带一个参数时写作 `@stdcall("libname_or_path")`,表示该无函数体 `extern fn` 从指定原生库导入且使用 `stdcall`; 该参数也可引用当前 `mod` 作用域中以字符串字面量初始化的 `let` 绑定。无参数时用于显式声明 `@fixed fn` 或 `@fixed` 方法使用 `stdcall`。 |
| `@fastcall` | 无函数体的 `extern fn` 声明前,或 `@fixed fn` / `@fixed` 方法定义前 | 带一个参数时写作 `@fastcall("libname_or_path")`,表示该无函数体 `extern fn` 从指定原生库导入且使用 `fastcall`; 该参数也可引用当前 `mod` 作用域中以字符串字面量初始化的 `let` 绑定。无参数时用于显式声明 `@fixed fn` 或 `@fixed` 方法使用 `fastcall`。 |
| `@bounded` | 仅允许出现在编译器生成的 `.fi` 接口文件中 | 表达公开 `let` 绑定的显式绑定元信息: `.fi` 不保留值本身,也不保留 `fn` 实现; 无参数形式标注公开 `let` 成员在声明初值阶段已绑定,带参数形式标注某个公开构造函数会在构造阶段绑定列出的公开 `let` 成员; 不属于普通 `.ff` 源码可手写语法。 |

## 5 模块系统

Feng 的模块系统负责源码组织、命名空间划分以及外部模块与二进制包导入,核心关键字为 `mod` 和 `use`。

本规范仅保留模块系统概要说明,详细的模块声明规则、模块导入规则和使用示例已拆分到独立文档: [Feng 语言模块系统规范](./feng-module.md)。

模块系统要点:

- 使用 `mod` 进行文件级模块声明,支持 `.` 分隔的多级命名空间。
- 模块默认等价于 `pr mod`,包外不可见; 需要对外暴露时必须显式使用 `pu mod`。
- 使用 `use` 导入源码模块、feng 自有二进制包和 C ABI 兼容包,并支持 `use ... as 别名`。
- 模块声明与模块导入在文件中有固定位置约束。
- 同一模块在合并后的模块级作用域内,不能出现重名 `type`; 顶层 `fn` 可在模块内部形成重载集合,返回值不参与重载,若参数类型在当前可见的显式契约关系下存在可重叠匹配则编译期报错。
- `use` 引入的公开 `type`、顶层 `fn` 与模块级 `let` / `var` 会进入当前模块可见作用域; 其中导入的 `fn` 不会与其他来源声明共同形成新的重载集合,导入的模块级 `let` / `var` 也不参与重载; 若形成重名,则编译期报错,应通过别名消除冲突。
- 模块级变量初始化规则区分“语言保证”和“编译器实现”: 模块级变量先按类型默认值建立存储,再按初始化顺序执行初始化表达式; 一个 `mod` 若 `use` 了其他 `mod`,则总是先初始化这些 `use` 目标,多个 `use` 按声明顺序执行,最后再初始化当前 `mod` 自身; 一个模块即使在多个位置被 `use`,也只会在首次需要时初始化一次; 同一 `mod` 若分布在多个文件中,跨文件的模块级 `let` / `var` 初始化顺序不受语言规范保障,编译器可自行采用稳定实现但不得作为对外承诺; 若在初始化过程中读取尚未完成初始化的模块级变量,则得到该变量当前值,初始情况下为类型默认值,`use` 循环也沿用这一规则。

## 6 C库来源与互操作规范

C 互操作负责 C 库来源声明、C 兼容类型映射、外部函数声明与 ABI 固定边界规则,核心语法包括调用方式注解、`extern fn` 和 `@fixed`。

本规范仅保留 C 互操作概要说明,详细的 C 库来源规则、C 兼容结构体/函数指针定义、调用方式注解、外部函数声明与完整示例已拆分到独立文档: [Feng 语言 C 互操作规范](./feng-interop.md)。

C 互操作要点:

- 使用 `@cdecl("libname_or_path")`、`@stdcall("libname_or_path")` 或 `@fastcall("libname_or_path")` 为无函数体的 `extern fn` 声明指定 C 库来源与调用方式; 参数也可引用当前 `mod` 作用域中以字符串字面量初始化的 `let` 绑定。
- 使用对象形式的 `@fixed type` 定义 C 兼容结构体或联合体,使用函数形式的 `@fixed type` 定义 C 兼容函数指针类型。
- 支持使用 `@union` 将对象形式的 `@fixed type` 声明为 C 联合体。
- 支持使用无参数的 `@cdecl`、`@stdcall` 和 `@fastcall` 显式声明 `@fixed fn` 定义的 C 函数调用方式; 未显式标注时默认使用 `cdecl`。
- 使用 `extern fn` 声明 C 外部函数,使用 `@fixed fn` 定义可传给 C 的 Feng 回调函数,并使用 `pu @fixed fn` 定义公开的 C ABI 导出函数。
- 编译器对 C 互操作相关声明执行 ABI 与类型安全检查。

## 7 类型系统

Feng 的类型系统采用强类型、静态类型设计,统一使用 `type` 定义具名类型,并通过 `@fixed` 标记其中进入 C ABI 的结构体、联合体和函数类型,同时支持类型推导、`spec` 契约、`fit` 适配、默认零值初始化与数组类型。

本规范仅保留类型系统概要说明,详细的数据类型、变量初始化、函数类型和自定义类型规则已拆分到独立文档: [Feng 语言类型系统规范](./feng-type.md); `spec` 契约与 `fit` 适配规则见 [Feng 语言 `spec` 规范](./feng-spec.md) 与 [Feng 语言 `fit` 规范](./feng-fit.md); `let` / `var` 的绑定与作用域规则已拆分到独立文档: [Feng 语言变量绑定与作用域规范](./feng-binding.md); Feng 原生 `type` 的对象语义已拆分到独立文档: [Feng 语言对象模型规范](./feng-object.md)。

类型系统要点:

- 基础类型包含整数、浮点、`bool`、`string`、数组和 C 互操作指针类型,其中 `i64` 的别名是 `int`,`f64` 的别名是 `float`。
- 有初始值的变量支持类型推导,无初始值变量使用默认零值初始化。
- `let` 用于声明不可变绑定,`var` 用于声明可变绑定。
- 模块内变量、函数内局部变量和类型成员必须显式使用 `let` 或 `var`; 参数可省略,省略时默认按 `let` 处理。
- `let` 与 `var` 都遵循块作用域,子级块可屏蔽上层同名变量。
- Feng 原生 `type` 采用引用语义; 实例创建可按“成员声明初值 → 构造函数 → 对象字面量初始化”的顺序完成,对象字面量位于创建流程最后阶段; 构造函数中对 `self.<let成员>` 的直接赋值视为构造阶段显式绑定,因此该 `let` 成员后续不得再在对象字面量中重复绑定。
- Feng 原生函数类型与 C 兼容函数指针类型分离设计。
- Feng 原生 `type` 支持成员、构造函数、方法和访问控制,类型成员默认公开,也可显式通过 `pu/pr` 设置可见性。
- 契约匹配通过 `spec` 与 `fit` 显式表达; 只有显式建立契约关系的具体类型,才可在要求该 `spec` 的位置上使用。

## 8 函数定义规则

Feng 的函数系统覆盖内部普通函数、程序入口函数、Lambda 表达式与闭包,核心关键字为 `fn`。

本规范仅保留函数系统概要说明,详细的普通函数规则、入口函数签名、Lambda 与闭包说明已拆分到独立文档: [Feng 语言函数系统规范](./feng-function.md); 参数的 `let` / `var` 默认规则见 [Feng 语言变量绑定与作用域规范](./feng-binding.md)。

函数系统要点:

- 使用 `fn` 定义内部普通函数、成员方法和构造函数; 模块级顶层 `fn`、成员方法与构造函数都可在各自声明作用域内按“名称 + 参数个数 + 参数类型”形成重载,若参数类型在当前可见的显式契约关系下可能同时匹配同一调用则编译期报错; 当重载函数或重载方法以函数值形式使用时,必须存在足以唯一确定目标函数类型的上下文,否则编译期报错。
- 函数与成员方法遵循 `pr` / `pu` 可见性控制。
- 函数参数必须显式标注类型,可省略 `let` / `var`; 省略时默认等价于 `let`,需要在函数体内修改参数时必须显式写为 `var`。
- 程序唯一入口为 `main(args: string[])`。
- `Lambda` 是函数实现或函数字面量的简写形式,不是独立的函数类型定义。
- 闭包支持捕获外部变量。

## 14 对象模型

Feng 原生 `type` 采用托管引用语义,与 `@fixed type` 表示的 C 兼容值类型明确分离。

对象模型要点:

- Feng 原生 `type` 实例在赋值、传参和返回时默认复制引用而不是复制对象本体。
- 支持构造函数调用 `Type(args)`、无参构造后的对象字面量 `Type { field: value }` / `Type() { field: value }`、以及构造后对象字面量 `Type(args) { field: value }` 等创建语法; 对 Feng 原生 `type` 而言,实例创建可按“成员声明初值 → 构造函数 → 对象字面量初始化”的顺序完成,对象字面量中的 `let` 成员只有在前置阶段尚未完成显式绑定时才可初始化; 若一个 `type` 未显式声明任何构造函数,则视为存在默认的公开无参构造函数。

## 17 完整代码示例

### 17.1 c_interop.ff

```feng
pu mod libc.interop;

// C 兼容结构体
@fixed
type Point {
    var x: int;
    var y: int;
}

// C 兼容函数指针类型
@fixed
type PointCB(p: Point);

let point_lib = "./libpoint.so";

// 声明 C 外部函数
@cdecl(point_lib)
extern fn point_add(p1: Point, p2: Point): Point;

@cdecl(point_lib)
extern fn exec_point_cb(p: Point, cb: PointCB);

// 定义 feng 回调函数
@fixed
fn on_point(p: Point) {
    print("Point:x=", p.x, " y=", p.y);
}
```

### 17.2 main.ff

```feng
mod main;
use libc.interop;
use mylib; // 导入 feng 自有二进制包

// feng 内部函数类型
type IntToInt(x: int): int;

// feng 内部普通函数
fn make_adder(base: int): IntToInt {
    return (x: int) -> base + x;
}

fn main(args: string[]) {
    // 操作 C 兼容结构体
    let p1 = Point {x: 10, y: 20};
    let p2 = Point {x: 5, y: 5};
    let res_p = point_add(p1, p2);
    print(res_p.x, res_p.y);

    // 传递 feng 回调给 C
    exec_point_cb(res_p, on_point);

    // 调用 feng 自有包成员
    let user = User{name:"test", age:20};
    add_user(user);

    // 闭包调用
    let add5 = make_adder(5);
    print(add5(3));

    // 数组操作
    let arr = [10, 20, 30];
    print(arr[0]);

    // 异常处理
    try {
        let num = arr[10];
    } catch {
        print("数组越界异常");
    } finally {
        print("程序执行完毕");
    }
}
```

## 18 核心语法约束

### 18.1 语言与类型约束

- 强类型、静态类型,无隐式类型转换。
- 支持 `spec` 契约、`fit` 适配、`Lambda` 与闭包,闭包不可用于 C 互操作。
- 类型系统统一采用 `type` 体系,并通过 `@fixed` 标记 C 兼容布局与函数类型,详细规则见 [feng-type.md](./feng-type.md)。
- 表达式具有确定的优先级与从左到右求值顺序,赋值使用独立语句规则。
- 支持 `throw` 与 `try/catch/finally` 结构化异常处理。
- Feng 原生 `type` 采用托管引用语义,`@fixed type` 采用 C 兼容值布局。
- 模块声明在顶部,`use` 段位于其下。

### 18.2 C互操作与分发约束

- `@fixed type`: 定义 C 兼容结构体、联合体和 C 函数指针类型
- `@cdecl("libname_or_path")` / `@stdcall("libname_or_path")` / `@fastcall("libname_or_path")`: 为无函数体的 `extern fn` 声明指定原生库来源与调用方式; 参数也可引用当前 `mod` 作用域中以字符串字面量初始化的 `let` 绑定
- `@union`: 将对象形式的 `@fixed type` 声明为 C 联合体
- 无参数 `@cdecl` / `@stdcall` / `@fastcall`: 声明 `@fixed fn` 定义的 C 函数调用方式,未显式标注时默认使用 `cdecl`
- `extern fn`: 声明 C 外部函数
- `@fixed fn` / `pu @fixed fn`: 定义可传 C 的 Feng 回调或定义 C ABI 兼容导出函数
- 异常不得跨越 `@fixed` ABI 边界传播
- 编译器对 `extern fn` 与 `@fixed` 标记内容做 C ABI 编译期检查
- Feng 普通 `fn` 与 C ABI 完全隔离,保障安全
- 程序入口 `main` 带 `string[] args` 参数,无需 `extern`
- 自动内存管理机制不接管 `@fixed` ABI 值内存
- 支持统一的 `.fb` 包格式,可按需要同时携带 feng 自有 ABI 层与 C ABI 层
- 二进制包不含源码,支持闭源复用与编译加速
- `.fb` 包可仅用于 feng 间复用,也可用于跨语言复用
