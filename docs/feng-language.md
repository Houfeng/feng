# Feng 语言核心规范

## 1 设计哲学

Feng 是一门**强类型、静态类型、支持鸭子类型(鸭型)**的编程语言,秉持极简主义设计理念,语法紧凑、关键字稀少,兼顾脚本语言的简洁性与 C 语言的高效性; 支持闭包、函数式编程与结构化异常处理,全程保障内存安全; 以 `type` 统一定义所有类型,通过 `extern` 标记实现与 C 语言的安全互操作,底层可无缝映射为 C 语言代码,无额外运行时开销; 搭配全自动 GC 实现无感知内存管理,支持 C 标准库、第三方 C 库的结构体和函数指针双向互操作,编译器通过 `extern` 标记完成编译期 ABI 与类型安全检查; 同时支持 feng 自有闭源二进制包分发和 C ABI 兼容包分发,兼顾源码保护、编译加速与跨语言复用。

## 2 语言核心特性

- 静态类型: 变量、参数、返回值类型在编译期确定,运行期不可更改。
- 强类型: 严格类型检查,禁止隐式类型转换,类型不匹配直接编译报错。
- 无 `any` 类型: 不提供动态任意类型,所有变量、参数均为明确类型。
- 鸭子类型(鸭型): 类型无需显式继承或实现,兼容性只按公开成员与公开方法签名在编译期检查,且同包与跨包规则一致。
- 支持闭包: `Lambda` 表达式可捕获外部作用域变量,延长变量生命周期。
- 类型推导: 变量有初始值时,可省略类型标注,编译器自动推导; 但若初始值是重载函数或重载方法值,且当前上下文无法唯一确定目标函数类型,则编译期报错。
- 表达式规则: 提供确定的运算符优先级、从左到右求值顺序、参考 C 的数值运算规则与无隐式转换语义。
- 函数返回值: `void` 代表空无类型,无返回值时可省略,有返回值可自动推导类型。
- 绑定规则: `let` 用于声明不可变绑定,`var` 用于声明可变绑定; 参数可省略 `let` / `var`,省略时默认按 `let` 处理,模块内变量、函数内变量和类型成员必须显式书写 `let` 或 `var`。
- 类型使用规范: 具名类型声明必须位于模块级顶层,不允许嵌套声明,不支持匿名类型定义,也不支持在类型位置直接内联声明类型; 同一模块内的 Feng 原生 `type` 与函数类型允许前向引用和互引用。
- 数组支持: 采用 `T[]` 语法,支持一维及多维数组,如 `T[][]`、`T[][][]`。
- 异常处理: 支持 `throw` 与完整 `try/catch/finally` 结构化异常处理,异常不得穿越 C ABI 边界。
- 默认零值: 未初始化变量由编译器自动赋予对应类型默认零值。
- 对象模型: Feng 原生 `type` 采用 GC 托管引用语义,对象字面量与构造函数并存。
- 内存管理: 全自动 GC,无指针、无手动内存分配或释放,并明确区分托管对象与 `extern` 边界上的非托管对象。
- C 互操作核心: `extern type` 定义可与 C 互通的类型,支持通过 `@union` 声明 C 联合体; `extern fn` 可声明 C 外部函数、定义可传 C 的 Feng 回调以及定义 C ABI 兼容导出函数,`link` 负责链接 C 库,全程进行编译期安全检查。
- 包分发特性: 支持 feng 自有二进制包(非 `extern` 公有成员)和 C ABI 兼容包双分发模式,支持闭源复用与编译加速。

## 3 基础语法规则

1. 文件扩展名: `.f`
2. 语句结束符: 分号 `;`
3. 代码块: 统一使用花括号 `{}` 包裹
4. 注释规范:
   - 单行注释: `// 注释内容`
   - 多行注释: `/* 注释内容 */`
5. 代码书写顺序: 模块声明 → 模块导入 → 链接指令 → 类型定义 → 函数/业务代码,不可颠倒; `link` 声明必须位于 `mod` / `use` 段之后、其他语句之前,建议上下各保留一行空行

## 4 核心关键字

| 关键字 | 含义 |
| --- | --- |
| `type` | 定义 feng 原生内部类型,包括对象类型和函数类型。 |
| `extern type` | 定义可与 C 语言互通的类型,包括 C 兼容结构体、联合体和 C 函数指针类型。 |
| `fn` | 定义 feng 内部普通函数、成员方法和构造函数。 |
| `extern fn` | 声明 C 语言实现的外部函数; 定义可作为函数指针传给 C 的 feng 回调函数; 定义 C ABI 兼容导出函数。 |
| `var` | 修饰可变变量、成员、参数。 |
| `let` | 修饰只读变量、成员。 |
| `pu` | 设置公开访问权限。 |
| `pr` | 设置私有访问权限。 |
| `self` | 类型内部引用自身,等价于 `this`。 |
| `mod` | 文件级模块声明。 |
| `use` | 导入外部公开模块或二进制包。 |
| `link` | 链接 C 标准库、第三方 C 库或 feng 二进制编译产物。 |

## 5 模块系统

Feng 的模块系统负责源码组织、命名空间划分以及外部模块与二进制包导入,核心关键字为 `mod` 和 `use`。

本规范仅保留模块系统概要说明,详细的模块声明规则、模块导入规则和使用示例已拆分到独立文档: [Feng 语言模块系统规范](./feng-module.md)。

模块系统要点:

- 使用 `mod` 进行文件级模块声明,支持 `.` 分隔的多级命名空间。
- 模块默认等价于 `pr mod`,包外不可见; 需要对外暴露时必须显式使用 `pu mod`。
- 使用 `use` 导入源码模块、feng 自有二进制包和 C ABI 兼容包,并支持 `use ... as 别名`。
- 模块声明与模块导入在文件中有固定位置约束。
- 同一模块在合并后的模块级作用域内,不能出现重名 `type`; 顶层 `fn` 可在模块内部形成重载集合,返回值不参与重载,若参数类型在鸭型规则下存在可重叠匹配则编译期报错。
- `use` 引入的公开 `type` 与顶层 `fn` 进入当前模块可见作用域时,不会与其他来源声明共同形成新的重载集合; 若形成重名,则编译期报错,应通过别名消除冲突。
- 模块级变量初始化规则区分“语言保证”和“编译器实现”: 模块级变量先按类型默认值建立存储,再按初始化顺序执行初始化表达式; 一个 `mod` 若 `use` 了其他 `mod`,则总是先初始化这些 `use` 目标,多个 `use` 按声明顺序执行,最后再初始化当前 `mod` 自身; 一个模块即使在多个位置被 `use`,也只会在首次需要时初始化一次; 同一 `mod` 若分布在多个文件中,跨文件的模块级 `let` / `var` 初始化顺序不受语言规范保障,编译器可自行采用稳定实现但不得作为对外承诺; 若在初始化过程中读取尚未完成初始化的模块级变量,则得到该变量当前值,初始情况下为类型默认值,`use` 循环也沿用这一规则。

## 6 C库链接与互操作规范

C 互操作负责 C 库链接、C 兼容类型映射、外部函数声明与 feng 回调定义,核心关键字为 `link`、`extern type`、`extern fn` 以及调用方式注解。

本规范仅保留 C 互操作概要说明,详细的 C 库链接规则、C 兼容结构体/函数指针定义、调用方式注解、外部函数声明与完整示例已拆分到独立文档: [Feng 语言 C 互操作规范](./feng-interop.md)。

C 互操作要点:

- 使用 `link` 链接 C 标准库、第三方 C 库和相关二进制产物。
- 使用 `extern type` 定义 C 兼容结构体、联合体和 C 函数指针类型。
- 支持使用 `@union` 将对象形式的 `extern type` 声明为 C 联合体。
- 支持使用 `@cdecl`、`@stdcall` 和 `@fastcall` 显式声明 C 函数调用方式; 未显式标注时默认使用 `cdecl`。
- 使用 `extern fn` 声明 C 外部函数,定义可传给 C 的 Feng 回调函数,以及定义公开的 C ABI 导出函数。
- 编译器对 C 互操作相关声明执行 ABI 与类型安全检查。

## 7 类型系统

Feng 的类型系统采用强类型、静态类型设计,统一使用 `type` 和 `extern type` 描述 feng 原生类型与 C 兼容类型,并支持类型推导、鸭子类型(鸭型)、默认零值初始化与数组类型。

本规范仅保留类型系统概要说明,详细的数据类型、变量初始化、函数类型、自定义类型和鸭型规则已拆分到独立文档: [Feng 语言类型系统规范](./feng-type.md); `let` / `var` 的绑定与作用域规则已拆分到独立文档: [Feng 语言变量绑定与作用域规范](./feng-binding.md); Feng 原生 `type` 的对象语义已拆分到独立文档: [Feng 语言对象模型规范](./feng-object.md)。

类型系统要点:

- 基础类型包含整数、浮点、`bool`、`string`、数组和 C 互操作指针类型,其中 `i64` 的别名是 `int`,`f64` 的别名是 `float`。
- 有初始值的变量支持类型推导,无初始值变量使用默认零值初始化。
- `let` 用于声明不可变绑定,`var` 用于声明可变绑定。
- 模块内变量、函数内局部变量和类型成员必须显式使用 `let` 或 `var`; 参数可省略,省略时默认按 `let` 处理。
- `let` 与 `var` 都遵循块作用域,子级块可屏蔽上层同名变量。
- Feng 原生 `type` 采用引用语义; 实例创建可按“成员声明初值 → 构造函数 → 对象字面量初始化”的顺序完成,对象字面量位于创建流程最后阶段,`let` 成员若已在更早阶段完成显式绑定则不得再次绑定。
- Feng 原生函数类型与 C 兼容函数指针类型分离设计。
- Feng 原生 `type` 支持成员、构造函数、方法和访问控制,类型成员默认公开,也可显式通过 `pu/pr` 设置可见性。
- 鸭子类型兼容性由编译器在编译期按公开成员进行检查,且同包与跨包规则一致。

## 8 函数定义规则

Feng 的函数系统覆盖内部普通函数、程序入口函数、Lambda 表达式与闭包,核心关键字为 `fn`。

本规范仅保留函数系统概要说明,详细的普通函数规则、入口函数签名、Lambda 与闭包说明已拆分到独立文档: [Feng 语言函数系统规范](./feng-function.md); 参数的 `let` / `var` 默认规则见 [Feng 语言变量绑定与作用域规范](./feng-binding.md)。

函数系统要点:

- 使用 `fn` 定义内部普通函数、成员方法和构造函数; 模块级顶层 `fn`、成员方法与构造函数都可在各自声明作用域内按“名称 + 参数个数 + 参数类型”形成重载,若参数类型在鸭型规则下存在可重叠匹配则编译期报错; 当重载函数或重载方法以函数值形式使用时,必须存在足以唯一确定目标函数类型的上下文,否则编译期报错。
- 函数与成员方法遵循 `pr` / `pu` 可见性控制。
- 函数参数必须显式标注类型,可省略 `let` / `var`; 省略时默认等价于 `let`,需要在函数体内修改参数时必须显式写为 `var`。
- 程序唯一入口为 `main(args: string[])`。
- `Lambda` 是函数实现或函数字面量的简写形式,不是独立的函数类型定义。
- 闭包支持捕获外部变量。

## 9 表达式与运算规则

Feng 的表达式系统覆盖字面量、调用、成员访问、下标访问、一元/二元运算与赋值语句,并为编译器提供确定的求值顺序与优先级模型。

本规范仅保留表达式概要说明,详细的表达式类别、运算符优先级、求值顺序、聚合字面量与赋值规则已拆分到独立文档: [Feng 语言表达式与运算规范](./feng-expression.md)。

表达式系统要点:

- 二元运算、函数调用参数和后缀访问均按从左到右顺序求值。
- `&&` 与 `||` 使用短路求值。
- `=` 是赋值语句,不作为普通表达式参与优先级运算。
- 数值运算参考 C,但不同数值类型之间必须显式转换后才能运算。
- 显式数值转换使用 C 风格 `(目标类型)表达式` 语法。
- `type` 对象与动态数组的 `==` / `!=` 默认比较引用身份。

## 10 Lambda表达式与闭包

`Lambda` 与闭包属于函数系统的一部分,本规范中不再重复展开。详细的 `Lambda` 语法、闭包捕获规则和示例见 [Feng 语言函数系统规范](./feng-function.md)。

## 11 数组与多维数组

- 语法: 一维 `T[]`、二维 `T[][]`、三维 `T[][][]`,以此类推
- 数组字面量使用 `[]` 包裹,多维数组采用嵌套写法
- 通过 `数组[下标]` 访问元素,下标从 `0` 开始,越界会触发运行时异常
- 动态数组为 feng GC 托管类型,不可用于 C 互操作

```feng
// 一维数组
let arr = [1, 2, 3];

// 二维数组
let mat = [[1, 2], [3, 4]];

// 数组元素访问
let num = arr[0];
let val = mat[1][1];
```

## 12 流程控制

Feng 的流程控制覆盖条件判断、模式匹配、循环与结构化异常处理。

本规范仅保留流程控制概要说明,详细的条件分支、`if` 表达式、模式匹配、循环与控制转移规则已拆分到独立文档: [Feng 语言流程控制规范](./feng-flow.md)。

流程控制要点:

- 支持 `if / else if / else` 条件分支。
- 支持 `if` 作为表达式进行条件求值与顺序匹配赋值。
- 支持 `while`、`for`、`break` 和 `continue`。
- `finally` 会在 `return`、`break`、`continue` 和异常展开前执行。

## 13 异常模型

Feng 的异常模型使用 `throw` 与 `try/catch/finally` 组合实现结构化错误传播。

本规范仅保留异常模型概要说明,详细的 `throw`、异常传播、`finally` 执行语义以及 C ABI 边界限制已拆分到独立文档: [Feng 语言异常模型规范](./feng-exception.md)。

异常模型要点:

- 使用 `throw` 显式抛出异常。
- 当前版本仅支持无绑定形式的 `catch { ... }` 捕获任意 Feng 异常。
- `finally` 中禁止再次执行新的控制流跳转语句。
- 未捕获异常不得穿越 `extern fn` 的 ABI 边界。

## 14 对象模型

Feng 原生 `type` 采用 GC 托管引用语义,与 C 兼容值类型 `extern type` 明确分离。

本规范仅保留对象模型概要说明,详细的实例创建、对象字面量、构造函数、引用语义与成员可变性规则已拆分到独立文档: [Feng 语言对象模型规范](./feng-object.md)。

对象模型要点:

- Feng 原生 `type` 实例在赋值、传参和返回时默认复制引用而不是复制对象本体。
- 支持构造函数调用 `Type(args)`、无参构造后的对象字面量 `Type { field: value }` / `Type() { field: value }`、以及构造后对象字面量 `Type(args) { field: value }` 等创建语法; 对 Feng 原生 `type` 而言,实例创建可按“成员声明初值 → 构造函数 → 对象字面量初始化”的顺序完成,对象字面量中的 `let` 成员只有在前置阶段尚未完成显式绑定时才可初始化; 若一个 `type` 未显式声明任何构造函数,则视为存在默认的公开无参构造函数。
- `let` 约束绑定是否可重新赋值,成员是否可修改由成员自身的 `let` / `var` 决定。
- `self` 只在定义于 `type` 中的成员 `fn` 或构造函数内合法; 无论直接调用还是作为方法值传递,都始终稳定绑定到对应的类型实例,而未定义在 `type` 中的普通 `fn` 或 `Lambda` 若直接使用 `self`,则编译期报错。
- Feng 原生对象上的 `==` / `!=` 默认比较引用身份。

## 15 垃圾回收GC

Feng 使用全自动垃圾回收机制管理原生运行时对象,并与 C 互操作边界上的非托管内存严格分离。

本规范仅保留 GC 概要说明,详细的托管对象范围、根集合、可达性、闭包捕获与 C 边界限制已拆分到独立文档: [Feng 语言 GC 与内存模型规范](./feng-gc.md)。

GC 要点:

- GC 托管对象包括字符串、动态数组、Feng 原生 `type` 对象与闭包捕获环境。
- `extern type`、`*T` 指针与 C 侧分配内存不受 GC 管理。
- GC 以可达性为准,正确处理对象图和循环引用。
- 当前语言版本不提供内建 pin、borrow 或析构器机制。

## 16 Feng 包分发与二进制复用规范

Feng 支持 feng 自有二进制包 `.fp` 与 C ABI 兼容二进制包 `.fcp` 两种标准化包分发模式,用于满足 feng 内部复用与跨语言复用两类场景。

本规范仅保留包系统概要说明,详细的包格式、清单文件、接口文件、编译与使用流程、约束规则和导入语法已拆分到独立文档: [Feng 语言包分发与二进制复用规范](./feng-package.md)。

包系统要点:

- 支持 `.fp` 与 `.fcp` 两种标准化包格式。
- `.fp` 适用于 feng 内部公有成员的二进制复用,不面向 C 语言调用。
- `.fcp` 适用于公开 `extern` 接口的跨语言分发与复用。
- 包系统支持闭源分发、编译加速和跨平台二进制产物组织。

## 17 完整代码示例

### 17.1 c_interop.f

```feng
pu mod libc.interop;

link "m";

// C 兼容结构体
extern type Point {
    var x: int;
    var y: int;
}

// C 兼容函数指针类型
extern type PointCB(p: Point);

// 声明 C 外部函数
extern fn point_add(p1: Point, p2: Point): Point;
extern fn exec_point_cb(p: Point, cb: PointCB);

// 定义 feng 回调函数
extern fn on_point(p: Point) {
    print("Point:x=", p.x, " y=", p.y);
}
```

### 17.2 main.f

## 18 核心语法约束

### 18.1 语言与类型约束

- 强类型、静态类型,无隐式类型转换。
- 支持鸭子类型(鸭型)、`Lambda` 与闭包,闭包不可用于 C 互操作。
- 类型系统统一采用 `type` / `extern type` 体系,详细规则见 [feng-type.md](./feng-type.md)。
- 表达式具有确定的优先级与从左到右求值顺序,赋值使用独立语句规则。
- 支持 `throw` 与 `try/catch/finally` 结构化异常处理。
- Feng 原生 `type` 采用 GC 托管引用语义,`extern type` 采用 C 兼容值布局。
- 模块声明在顶部,`use` 段位于其下; `link` 声明必须位于 `mod` / `use` 段之后、其他语句之前,建议上下各保留一行空行。

### 18.2 C互操作与分发约束

- `extern type`: 定义 C 兼容结构体、联合体和 C 函数指针类型
- `@union`: 将对象形式的 `extern type` 声明为 C 联合体
- `@cdecl` / `@stdcall` / `@fastcall`: 声明 C 函数调用方式,未显式标注时默认使用 `cdecl`
- `extern fn`: 声明 C 外部函数、定义可传 C 的 Feng 回调或定义 C ABI 兼容导出函数
- 异常不得跨越 `extern fn` 边界传播
- 编译器对 `extern` 标记内容做 C ABI 编译期检查
- Feng 普通 `fn` 与 C ABI 完全隔离,保障安全
- 程序入口 `main` 带 `string[] args` 参数,无需 `extern`
- 全自动 GC,不管理 C 兼容类型内存
- 支持 `.fp`(feng 自有包)和 `.fcp`(C ABI 包)双分发模式
- 二进制包不含源码,支持闭源复用与编译加速
- `.fp` 包仅 feng 间复用,`.fcp` 包支持跨语言复用
